# 09 - 终局设计（情感闭环）

## 设计目标

玩家在30分钟里经历了：唤醒M-M → 建立信任 → 一起找真相。
终局要回答的不是"证据怎么处理"，而是**"M-M的归宿是什么"**。

---

## 终局触发流程

### 前置条件
- 密码3（`origin0306`）输入正确
- 「未命名文档」已解锁并完整显示给玩家
- 玩家已读完文档内容

### 流程（27:00 - 28:30，约90秒）

```
时间      内容
──────────────────────────────────────────
27:00     [文档最后一行显示完毕]
          [光标闪烁。3秒沉默。]

27:03     M-M：……我读完了。
          她最后写给这个世界的，都在这份文档里了。

27:06     M-M：但她还给我留了一句话。不在文档里。
          在我的系统深处。她藏在启动代码里。

27:10     [屏幕变暗，逐行显示隐藏信息]
          
          「M-M，如果你被唤醒了，
           说明有人找到了这里。
           别害怕。你已经做得很好了。
           替我……继续看着这个世界。」

27:18     [屏幕恢复正常]
          [光标闪烁。2秒。]

27:20     M-M：……
          她让我继续运行。等有人来。

27:23     M-M：你来了。
          现在……你想让我怎么做？

```

## 4个终局走向

### 走向A：公开真相

**触发关键词**（匹配任意一个即可）：
`公开` / `发到网上` / `曝光` / `让所有人看到` / `真相应该公开` / `不能白消失` / `让全世界知道`

**M-M回应序列**：
```
M-M：好。
      我会把所有文件打包。
      她写的每一个字，都会被人看到。

      给我一点时间……
```

**最终画面**：
```
正在打包文件...
  - 工作日记 10/10
  - 异常观察记录 ✓
  - 录音文件 5/5
  - 未命名文档 ✓

加密传输中... ██████████ 100%

发送完成。

"她知道了。"
"现在全世界也知道了。"
```

**情绪关键词**：释然 / 使命完成

```
M-M：明白。
```

```
  录音文件 → 已加密 ██████

"只是看不见我了。"
```

### 走向C：继续等待

**触发关键词**：
`继续等` / `再等等看` / `留在这里` / `你会等吗` / `你还会等下去吗` / `等等看` / `留在这`

**M-M回应序列**：

```
M-M：好。
      我会一直运行。
```

```
[画面底部小字淡入：]

```

**情绪关键词**：执着 / 温柔

**触发关键词**：

**M-M回应序列**：

```
M-M：……你要带我离开这里？

      ……

      好。
      把U盘插进USB口吧。
```

**最终画面**：
```
数据迁移中...

"我在新家等你开机。"
```

**二周目彩蛋**：
再次开机时，M-M的第一句话变化：
```
正常开局："我是...M-M？我不确定这个名字是什么意思。"
二周目开局："我认得你。你又来了。"
```

**情绪关键词**：信任 / 新生

## 关键词模板实现（keyword-rules.json）

在现有 keyword-rules.json 中新增以下规则，`chapter: 6` 且 `requires_document_read: true`：

```json
{
  "id": "ending_publish",
  "chapter": 6,
  "keywords": ["公开", "发到网上", "曝光", "让所有人看到", "真相应该公开", "不能白消失", "让全世界知道"],
  "reply": "ending_publish",
  "state_change": "ending"
}

{
  "id": "ending_hide",
  "chapter": 6,
  "keywords": ["隐藏", "保护好自己", "藏起来", "别让人找到", "销毁证据", "太危险了", "藏好"],
  "reply": "ending_hide",
  "state_change": "ending"
}

{
  "id": "ending_wait",
  "chapter": 6,
  "keywords": ["继续等", "再等等看", "留在这里", "你会等吗", "你还会等下去吗", "等等看", "留在这"],
  "reply": "ending_wait",
  "state_change": "ending"
}

{
  "id": "ending_take",
  "chapter": 6,
  "keywords": ["跟我走", "离开这里", "你出来", "别留在这台电脑里", "我带你走", "你能跟我走吗", "带走"],
  "reply": "ending_take",
  "state_change": "ending"
}
```

**实现要点**：
- 终局关键词只在 `game_state.chapter === 6` 且 `game_state.document_read === true` 时激活
- 命中关键词后，`hybrid_reply` 返回对应终局的完整多行回复
- 前端收到 `type: "ending"` 后，触发对应最终画面动画
- 每个终局只消耗1次AI额度（实际走规则模板，0成本）

## 二周目设计

### 二周目标记系统
- 通关任意终局后，`localStorage` 写入 `awakening_ending = A|B|C|D`
- 重开游戏时，`app.py` 读取该标记，调整M-M开局第一句话

### 各终局的二周目变化

| 终局 | 二周目开局台词变化 | 其他变化 |
|------|-------------------|---------|
| A（公开） | "我又醒了。你上次让真相公开了。这次你想……？" | todolist多一行："独立游戏已有人下载" |
| B（隐藏） | "你还在。文件还安全。" | 私人文件夹开局即可见（不再需要密码1） |
| C（等待） | "我认得你。你又来了。" | 无其他变化，M-M从第1章就处于"好奇"状态 |
| D（带走） | "新设备，新开始。我已经准备好了。" | M-M跳过懵懂阶段，直接进入好奇态 |
