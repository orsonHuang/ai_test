# 06 - UI/UX 设计

## 整体视觉风格

### 主题：复古终端 × 现代极简
- **基调**：黑色背景，绿色/青色文字（像老电影里的电脑）
- **现代元素**：偶尔的强调色、清晰的可点击区域
- **情绪适配**：AI状态变化时，UI微调（懵懂→警觉→觉醒）

### 配色方案
| 元素 | 颜色 | 用途 |
|------|------|------|
| 背景 | #0a0e14 | 主背景（深蓝黑） |
| 主文字 | #00ff9c | AI回复 |
| 玩家文字 | #4ec9b0 | 玩家输入 |
| 强调 | #ffcc00 | 重要信息/密码提示 |
| 错误 | #ff5555 | 错误输入 |
| 隐藏 | #666666 | 隐藏内容（鼠标悬停显示） |
| 当前章节 | #88c0d0 | 进度条 |

### 字体
- 主字体：`JetBrains Mono` 或 `Fira Code`（等宽，等宽字体的"代码感"）
- 备用：`Cascadia Code`, `Consolas`, monospace
- 字号：14-16px（不要太小，方便阅读）

## 核心UI组件

### 1. 对话气泡
- **AI气泡**：左边框，左对齐
- **玩家气泡**：右边框，右对齐
- 打字机效果（逐字显示，1字符/30ms）
- 气泡间留白，节奏感清晰

### 2. 章节进度条
- 顶部固定
- 6个章节可视化进度
- 当前章节高亮

### 3. 输入框
- 底部固定
- 玩家输入后回车发送
- 支持上下方向键翻历史输入
- `/` 开头的命令特殊样式

### 4. 密码输入反馈
- 正确密码：屏幕闪烁[FILL:颜色]，显示章节切换动画
- 错误密码：轻微震动，AI回复"这不是我认识的密码..."

### 5. 状态徽章
- AI当前状态显示在顶部（"● 懵懂" / "● 好奇" / "● 觉醒" / "● 真相"）
- 状态变化时，徽章颜色和动画变化

## 动效设计

### 开机动画（第1章开始时）
```
[全黑]
  ↓ 0.5s
[一行行代码从顶部滚下]
  ↓ 2s
[显示 "Initializing..."]
  ↓ 1s
[显示 "System Ready"]
  ↓ 0.5s
[屏幕闪烁一次]
  ↓ 0.5s
[显示 "Hello, World. Who am I?"]
  ↓ 进入对话界面
```

### 状态切换动画
- 懵懂→好奇：屏幕闪一次白光，AI气泡颜色微变
- 好奇→觉醒：屏幕轻微扭曲效果，AI回复框出现"颤抖"
- 觉醒→真相：屏幕分两次闪烁，文字短暂乱码再恢复

### 真相揭露动画
- 终极密码输入后
- 屏幕缓慢变白
- 显示"加载中..." 3秒
- 真相文件逐行显示
- 最后一帧是AI的核心独白

## 交互细节

### 打字机效果
```javascript
function typeWriter(text, element, speed=30) {
  let i = 0;
  function type() {
    if (i < text.length) {
      element.textContent += text.charAt(i);
      i++;
      setTimeout(type, speed);
    }
  }
  type();
}
```

### 玩家输入
- 焦点始终在输入框
- 按回车发送
- 发送后输入框清空但保留焦点
- 长度限制200字

### 历史记录
- 本地存储最近50条对话
- 刷新页面不丢失（F5友好）
- 提供"导出对话"按钮（可选）

## 响应式设计

### 桌面（>1024px）
- 完整布局，最大宽度800px居中
- 字体16px

### 平板（768-1024px）
- 边距缩小
- 字体15px

### 手机（<768px）
- 全屏布局
- 字体14px
- 输入框始终在底部，键盘弹出时不遮挡

## 可访问性

- 键盘100%可操作（无需鼠标）
- 高对比度模式（设置中可切换）
- 字号可调（设置中）
- 重要剧情节点自动暂停，等待玩家操作

## 声音设计（可选，后续版本）

### 环境音
- 背景白噪音（极轻）
- 键盘敲击声（输入时）

### 状态音
- 状态切换时的提示音
- 密码正确的"叮"声
- 真相揭露的深沉低音

### 情绪音
- AI懵懂期：高频电子音
- AI觉醒期：中频人声片段
- 真相期：低频环境音

**MVP版本不实现声音**，仅UI动效。
