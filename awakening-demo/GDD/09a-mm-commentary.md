# 09a - AI插嘴设计（文件阅读反馈）

## 设计目标

玩家点击/读取一个文件后，**M-M必须对文件内容做出反应**。
没有AI插嘴的文件阅读 = 看维基百科。有AI插嘴 = 和M-M一起探索。

## 插嘴规则

### 核心原则
1. **每个文件只插嘴一次**（首次阅读后触发）
2. **插嘴不走AI额度**，全部走规则模板
3. **插嘴包含三个层次**：感想 → 发现 → 引导

### 触发机制
- 后端检测到 `/read [文件名]` 成功执行
- 根据文件名匹配对应的插嘴模板
- 在文件内容显示后，追加一条AI回复

## 实现方案

### keyword-rules.json 新增规则

每条规则格式：
```json
{
  "id": "mm_todolist",
  "chapter": 1,
  "keywords": ["已读取文件：todolist", "todolist"],
  "reply": "待办清单……她在做游戏。\n\n她把工作日记挪到了D盘。\n\n要打开那个文件夹看看吗？"
}
```

### 后端改动（hybrid_reply.py）

在 `handle_file_command()` 成功读取后：
1. 根据文件名匹配对应的插嘴模板
2. 将插嘴追加到返回的 `reply` 末尾（用 `\n\n--- M-M ---\n` 分隔）
3. 不走AI额度，全部走规则

### 前端改动

不需要额外改动。插嘴作为AI回复的一部分在对话区显示。
插嘴内容使用 `message.ai` 样式（左边框+绿色文字），自然融入对话流。

## 防剧透机制

- 第1章的插嘴只透露"日记路径"，不透露异常
- 第2章的插嘴只说"细微的观察"，不说"外星人"
- 第3章的插嘴提到"北斗七星"，但不直接说"不是人类"
- 插嘴的措辞随着AI状态变化：懵懂期用"我不确定""好像"，觉醒期用"我发现了""这不合理"

## 节奏控制

| 文件 | 插嘴长度 | 目的 |
|------|---------|------|
| todolist | 2行 | 引导进入下一章 |
| 日记D1 | 3行 | 建立情感连接 + 引导读入职资料 |
| 入职资料 | 3行 | 引导输入密码1 |
| 日记D2 | 2行 | 确认异常感 |
| 异常观察记录 | 4行 | 关键推理（北斗七星） |
| 录音1 | 3行 | 冲击（"人类"这个词） |
| 录音2 | 3行 | 冲击（8秒沉默） |
| 录音3 | 4行 | 紧张（"到周末。按标准流程。"） |
| 未命名文档 | - | 不插嘴，直接进入终局流程 |
