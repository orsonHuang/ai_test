# 测试用例：游戏流程

## 启动测试
1. 启动服务：`python app.py`
2. 浏览器打开：`http://localhost:8080`
3. 应看到开机动画 → 进入对话界面

## 基础对话测试

### 测试1：识别关键词模板
- 输入：`你是谁`
- 期望：AI回复"我是... 我不太确定..."（rule_template类型）

### 测试2：识别文件建议
- 输入：`看看邮件`
- 期望：AI列出 emails 目录下的文件

### 测试3：列出文件
- 输入：`/files`
- 期望：列出所有可访问文件

### 测试4：读取文件
- 输入：`/read welcome.txt`
- 期望：显示欢迎语内容

### 测试5：AI状态查询
- 输入：`/status`
- 期望：显示当前章节、AI状态、AI调用次数

## 密码测试

### 测试6：第一个密码
- 输入：`alpha-7`
- 期望：触发密码识别，章节切到3，AI状态切到puzzled

### 测试7：终极密码
- 输入：`X-7-final`
- 期望：触发密码识别，章节切到6，AI状态切到truth

## 完整流程测试

### 阶段1（第1章）
- 玩家：开机 → AI懵懂
- 玩家：`/help` → 显示命令
- 玩家：`/read welcome.txt` → 读到欢迎语

### 阶段2（第2章）
- 玩家：`看看邮件` → AI列出邮件
- 玩家：`/read emails/1.md` → 读到第一封邮件
- 玩家：`你是谁` → 关键词模板回复

### 阶段3（第3章）
- 玩家：`alpha-7` → 触发密码，状态切换

### 阶段4-6
- 待GDD内容填充后测试

## 边界测试

### 测试8：空输入
- 输入：（直接回车）
- 期望：服务器返回"..."，不报错

### 测试9：超长输入
- 输入：201+字符
- 期望：前端input maxlength=200限制，后端正常处理

### 测试10：API失败
- 临时设置错误的DASHSCOPE_API_KEY
- 玩家输入长文本（触发AI）
- 期望：API失败时返回降级回复，不崩溃
