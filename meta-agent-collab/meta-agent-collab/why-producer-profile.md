# PRODUCER_PROFILE 的存在理由

> **状态**：stable（迭代中）
> **本文是什么**：回答一个基础性问题——"AI 不能凭对话理解人吗？为什么需要一份外置的用户档案？"
> **不是什么**：PROFILE 的写作指南（格式和内容边界见 `doc-ecosystem.md`）。

---

## 触发这篇的核心问题

框架里有 `PRODUCER_PROFILE.md`——一份描述"制作人（用户）是谁、偏好什么、怎么决策"的文件。直觉上这很奇怪：**AI 不是在每次对话里跟我聊吗，为什么还要一份文件来描述我？**

这个直觉错在：它假设"AI 跨对话保留对人的理解"。实际上 AI **每轮新对话都从空白启动**——和跨日记忆的道理一样。

---

## 三种失败模式（没有 PROFILE 时）

### 失败 1 · AI 对人的判断跨对话漂移

同一用户、同一项目，两次对话中 AI 对"这人偏好什么风格的输出"的判断可能完全不一致：

- 第一次对话：用户要求"精简、不啰嗦" → AI 学会了
- 第二天新对话：AI 不知道这个偏好 → 输出又变成默认啰嗦风格
- 用户再次纠正 → AI 再次学会
- 第三天再来一次……

这不是 AI 不听话，是**偏好信息没有被外置**。每次新对话，AI 对人从零开始建模。

### 失败 2 · 用户每次新对话要从零解释自己

没有 PROFILE 时，用户在新对话里要重新说一遍：
- "我喜欢短输出"
- "我不需要你解释每一步，直接做"
- "涉及钱/数据的事要先问我"

这些信息在单次对话里能传达——但**每个新对话都要重复一遍**。用户会累，最终放弃纠正，"随便吧"。

### 失败 3 · 长期偏好无法跨 AI 传递

如果用户今天用模型 A、明天用模型 B（或同一模型的不同版本），没有 PROFILE 时：
- 模型 A 对话里积累的对人的理解 → 模型 B 完全不知道
- 用户从"和一个懂我的 AI 协作"变成"和一个陌生人协作"
- 文件层的 PROFILE 解决了跨模型传递——**偏好写在文件里，哪个 AI 读都一样**

## PROFILE、memory、_observations 的三层关系：工作集 → 素材 → 结晶

很多人问：既然有 `_observations.md`（AI 每次对话追加观察素材），为什么还需要 PROFILE 和 memory？答案是三层各司其职：

| 维度 | memory | _observations | PRODUCER_PROFILE |
|------|--------|---------------|------------------|
| 本质 | **工作集**——近期活跃模式 | **素材**——流式记录 | **结晶**——稳定偏好 |
| 粒度 | 中（"用户最近 3 次都选了 X"） | 细（"用户今天说了 X"） | 粗（"用户偏好 Y 类型"） |
| 更新 | AI 每次对话结束 / 每次纠错 | AI 每次对话追加 | 用户拍板后入档（周度） |
| 读者 | AI（每次对话必读，≤100 行） | AI（复盘时读素材拼画像） | AI（按场景触发读取） |
| 可靠性 | 中（≥2 次触发，待验证） | 低（单次观察可能不准） | 高（经过多次验证和用户拍板） |
| 衰减 | 30 天未触发 → 降冷条目 | 不衰减 | 不衰减（旧版本保留） |

### 晋升管道：observations → memory → PROFILE

```
observations（素材·流式追加）
    ↓ 同类 observation 出现 ≥2 次
memory.md（工作集·≤100 行·即时但可能不准）
    ↓ 同类 pattern ≥3 次 + 周度 review 确认
PROFILE（结晶·版本化·精准但有滞后）
```

**这个管道的设计选择**：
- observations 到 memory 的晋升**不需要用户拍板**——AI 自己判断"这个 pattern 可能重复"，低成本写入 memory
- memory 到 PROFILE 的晋升**必须用户拍板**——因为 PROFILE 是"对用户人格的理解"，AI 无权单方面定义
- 灵感来源：self-improving skill 的 3 次确认晋升规则——消除了"什么时候该升级"的模糊性

---

## 什么时候不需要 PROFILE

PROFILE 不是必选项。以下场景可以不建：

- **协作频率低**——一个月聊一两次，建立画像的成本 > 收益
- **项目周期短**——一两周结束的项目，AI 不需要"长期理解"用户
- **用户角色单一**——你只是让 AI 做一件固定的事（"帮我翻译"），没有决策偏好需要记录
- **每次对话从零开始的用户自己接受**——不觉得重复解释是负担

**建不建的判断标准**：如果你发现自己在新对话中**反复纠正 AI 同一种行为**——那就是该建 PROFILE 的信号。

---

## 底线

PROFILE 不是为了"让 AI 认识我这个人"，而是为了**消除跨对话中的重复纠正**。它是一个工程优化——把"每轮都要解释"的信息从对话预算中移出，放到文件层一次性定义。
