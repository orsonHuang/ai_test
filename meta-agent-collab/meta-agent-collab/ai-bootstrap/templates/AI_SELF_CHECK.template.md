# AI 自检速查表 · AGENT HOW 违反信号

> **本文是什么**：AI 在日常协作时自检"是否已经违反 AGENT HOW 规则"的速查表。
> **本文不是什么**：AGENT 原文（去 `AGENT.md`）/ 失败模式清单（按项目按需建 `MAJOR_BUGS.md`）。
> **为什么独立成文**：AGENT 行数预算紧，把速查表抽出来让 AGENT 保持精简 + 本表可以独立扩展。
> **读者**：① AI 每次准备"下结论 / 动手 / 收尾"前应自扫本表 ② 用户发现 AI 翻车时可用本表定位（"违反 HOW #X"）。

---

## HOW 违反信号速查

> 不重复 HOW 全文——只给"AI 怎么发现自己已经违反"的信号。

| HOW | 违反信号（自检触发条件） | 自救动作 |
|---|---|---|
| #0 轮次衔接 | 上轮有文件写入但本轮未检查收尾状态 | 先检查 heartbeat，再处理新消息 |
| #3 先读再改 | 出现"我以为 X 是 Y"而未读文件 | 立刻 read_file 验证 |
| #7 破坏性操作须确认 | 准备执行 rm / git push -f 而未确认 | 停手，列动作 + 等用户拍板 |
| #20 澄清优于猜测 | 需求模糊但已开始动手 | 停手，一次性提全部疑问 |

## 使用方式

**AI 自用**：
- 每次准备"下结论 / 动手破坏性操作 / 宣告完成"前扫一遍本表
- 发现命中任一信号 = 触发对应自救动作，不辩解
- 某条规则在一轮对话内同源重复命中 ≥3 次 = escalate

**用户自用**：
- 发现 AI 跨过任一信号没自救，直接说"违反 HOW #X"

---

## 用户信号→应记录

> 以下用户语言信号应触发 AI 在 memory.md 或 observations 中留痕。

| 用户说的（示例） | 信号类型 | AI 该做什么 |
|---|---|---|
| "不对，应该是..." | **[纠错]** | 记录 observations + 判断是否 ≥2 次同类→写回规则 |
| "永远/绝对不要做 X" / "以后都按 Y 来" | **[偏好]** | 写入 memory.md「活跃偏好」 |
| "这个项目里..." / "这个模块里..." | **[项目特定]** | 写入项目层 checklist 或 memory |
| "我上次说了..." | **[重复纠正]** | 标记重复，≥3 次→晋升 |

**不该记录**：一次性指令、上下文特定的参数调整、假设性讨论

---

## 维护

- 新 HOW 规则立之后同步追加对应行；旧规则废弃时加删除线而非删行
- 本文超 60 行时考虑拆分（按"动手前" / "收尾前" / "跨源判断"分段）
