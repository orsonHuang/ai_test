# 活跃工作集（HOT · ≤100 行）

> **本文是什么**：介于 observations（素材）和 PROFILE（结晶）之间的即时工作层。
> 记录本轮/近期出现的可复用模式。3 次同类出现 → 提议晋升到 AGENT 或 PROFILE。
> **不是什么**：observations 的替代品（observations 只追加不删改）；PROFILE 的替代品（不版本化、不标证据强度）。
> **维护**：每次对话结束检查是否需要增/改/降级。30 天未触发的条目降为冷条目。

## 维护边界（该记什么 / 不该记什么）

| ✅ 适合记 | ❌ 不适合记 |
|----------|-----------|
| 长期身份信息（职位、团队、角色） | 一次性临时信息（"今天的临时配置"） |
| 稳定偏好（分点/表格/极简风格） | 敏感数据（密码、身份证号、密钥） |
| 长期项目背景与目标 | 频繁变动的状态（"当前 sprint day 3"） |
| 反复确认的禁用词与约定 | 只在某项目/某模块内有意义的细节 |
| 被用户明确说"以后都按 XXX 来"的偏好 | 技术上每周都会过期的信息 |

**维护动作**：
- 明确偏好："以后都按 XXX 来"→ 立刻写入活跃偏好
- 发现错误/过时：告诉 AI 更正，条目逐步修订
- 敏感场景：涉及密码/密钥/身份证时，主动告知用户不写入 memory
