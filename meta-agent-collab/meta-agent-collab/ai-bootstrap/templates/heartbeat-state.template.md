# 心跳状态

> **用途**：对话启动 + AI 输出结束时读取。由本文自主判断收尾状态，多一层保底。
> **维护**：每次 AI 输出结束时自动更新（AGENT §22）。不应手动编辑。

---

## 会话启动时（CodeBuddy 规则强制加载本文件时）

> 上轮对话是否正常收尾？检查「上次收尾状态」→ 未收尾 → 先补收尾再回应新消息。

---

## 输出结束时 · 自主判断（AI 每轮回复完成后必须执行）

> 本段由 heartbeat-state.md 自己承载判断逻辑——不依赖 AI 记住 AGENT §14。

1. 读「未 commit 改动」字段
2. **若 = 是**：
   - 本轮有 `replace_in_file` / `write_to_file`？→ 收尾条件成立
   - 用户本消息含「收尾 / 收工 / 完成 / 提交」？→ 收尾意图确认
   - 收尾流程是否已在本轮跑过？（看下方「wrap-up 完成」字段）
   - **若需要收尾且未跑过** → 立即执行 AGENT §14 收尾三件套（同步 + 沉淀 + 自检 + observations + NOW + memory），全部完成后最后一步 commit
   - **若已跑过** → 跳过，直接进入 commit 或下一动作
3. **若 = 否** → 无需动作，安全通过
