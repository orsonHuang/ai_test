## 工具使用软提示

- **定位型查询优先 `search_content` 而非 `read_file` 全文**：当目的是"某段现状 / 某规则是否存在 / 某指针指向哪"时，`search_content` 只回匹配行 + 少量上下文。但需要理解"结构性关系"（章节顺序 / 前后逻辑）时仍必须全读。
- **复杂替换前先 `search_content` 核对 `old_str` 字面形态**：触发条件——①多行 old_str ②含中英文混排 / 引号敏感段落 ③同文件已连续 ≥3 次 replace（记忆已漂移）。单行独特字符串不必。

---

## 对话输出软提示

> 管"AI 写给用户看的话"的冗余。

- **禁开场白**：不写"好的，我来帮你..." / "让我先..." 这类行动预告。直接进正文或直接发工具调用。
- **禁意图复述**：用户说"改 X 然后推送"，不要回"好的，我将改 X，然后推送"——无歧义时复述 = 纯浪费。例外：真有歧义时直接问具体问题。
- **禁收尾总结重写**：回答末尾不再把刚说的内容"总结一下"重写一遍。结构化出口（自检 / 交账 / NOW 同步）≠ 内容总结。
