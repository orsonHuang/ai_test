# 02 · 搭建顺序

> **本文是什么**：你（AI）问完 01 的问题、得出组合后，按本文对应路径建文件。
> **前置条件**：必须已完成 01 所有问题。如果你跳过 01 直接读这里，**停下，回去问问题**。

---

## 组合速查 · 你该走哪条路

01 问答结束后，你手里有对方的所有答案。按下面的映射判定组合：

| 关键答案 | 落在 |
|---------|------|
| Q8=不需要 且 Q9/Q10/Q11 全部不需要 | → **组合 A** |
| Q8=需要 但 Q9/Q10/Q11 全部不需要 | → **组合 B** |
| Q8=不需要 且 (Q9=需要 或 Q10=需要) | → **组合 C** |
| Q8=需要 且 (Q9=需要 或 Q10=需要 或 Q11=B) | → **组合 D** |
| Q8=需要 且 Q11=C（画像）| → **组合 E**（或含 Q12=A / Q13=A 的叠加） |

---

## 组合判定后 · 二次确认 + 叠加规则

> 组合速查表覆盖了主要路径，但总有边缘情况。以下规则处理所有漏掉的 case，**不增组合数**。

### 二次确认 · 解矛盾

| 情况 | 处理 |
|------|------|
| 组合装了 CHANGELOG 但 Q9=B（不要操作记录） | → 降为基础组合，CHANGELOG 当信号埋着 |
| 组合 C 装了 README 但对方做零碎活（海报/审阅等，非代码项目） | → 问一句"你需要项目 README 吗？"不要默认建 |
| Q8=B（不跨会话）但 Q11=B（要活跃记忆） | → 活跃记忆依赖跨会话机制，把活跃记忆当信号埋着，等 Q8 条件满足再触发 |

> 核心原则：**用户的 Q 答案优先级高于组合默认值**。组合和 Q 答案冲突时，Q 答案胜出。

### 叠加规则 · 不改变基础组合的零件

以下零件不改变基础组合编号，但按 Q 答案直接加到**任何组合**上：

| 条件 | 零件 | 安装位置 |
|------|------|---------|
| Q12=A | `AI_SELF_CHECK.md` | 工作区根目录，建骨架（当前规则少就先列已有的 4-5 条） |
| Q13=A | `AI_SOFT_HINTS.md` | 工作区根目录，内容几乎普世，不需大幅改写 |
| Q14=A | `MAJOR_BUGS.md` | 工作区根目录，建骨架（现象 + 根因 + 修复 + 为什么上次没发现） |

**举例**：用户选了组合 A → 但 Q12=A, Q13=A → 实际产出 = 组合 A 的 3 个文件 + AI_SELF_CHECK + AI_SOFT_HINTS，共 5 个文件。信号嵌入段也相应更新，删掉这两条的埋点（已经建了就不用埋了）。

### 判定后输出格式

向用户报告时，用两段式：

```
## 组合判定：B（基础版 + 跨会话连续性）

| 会建的 | 不会建的（信号埋着） |
|--------|---------------------|
| AGENT.md | CHANGELOG |
| ... | ... |

✚ 叠加（按你的选择）：
- Q12=A → 加 AI_SELF_CHECK.md
```

## 通用前置 · 所有组合第一步

### 确定工作区根目录

**如果 Q1 答 A（从零开始）**：
- 和对方商量工作区根目录的名字。**不要替 ta 取**——这是 ta 工作区的"姓"
- **推荐思路**：ta 的名字 / 代号 / 一个有意义的词。原作者用自己名字
- **避免**：`workspace/`、`my-project/`、`dev/` 这种通用词——一年后 ta 自己都不记得这是什么

**如果 Q1 答 B/C（已有项目）**：
- 迁入方案 3 选 1，让对方自己选：
  - **B1 · 物理搬进新工作区**：建新目录 → `mv` 项目进去。代价：IDE 路径要改
  - **B2 · 工作区 = 项目父目录**：不动项目，把元层文件建在项目父目录下
  - **B3 · 符号链接**：新工作区里 `ln -s` 指过去。仅在对方明确说"我懂 symlink 且项目不能动"时推
- **迁入场景额外提醒**：整体耗时多 10-15 分钟（扫现有项目 + 选方案 + 处理已有文档）

**通用收尾**：工作区根路径告知 AI，后续所有文件在这个根下操作。

### Step A1 · 建 AGENT.md（全局 · 含路由规则）

- **模板**：`templates/AGENT.template.md`
- **关键动作**：
  - 保留顶部哲学行（100 行内 / WHAT-WHY-HOW / 犯错即记录）
  - WHAT 段写"跨项目通用协作偏好"，明确列"不在本文件出现"的内容
  - **WHAT 段顶部加称呼信息**（前置·互相认识一中收集的）：
    ```
    > 用户称呼：[name]
    > AI 称呼：[name]（用户对 AI 的称呼）
    ```
    如 AI 称呼未取，第二行不写。
  - HOW 段按 Q4/Q5 的原话填规则（5-8 条）
  - 如果 Q7 选了预防性规则，按 Q7 答案塞入 0-11 条
  - **HOW 段尾部追加 AI 称呼行为约束规则**（如果用户给 AI 起了名）：
    ```
    N. **AI 不以此称呼自称**：用户给 AI 起的名字仅用于识别呼叫（用户说"[AI名]，改一下"时 AI 知道在叫自己），AI 自称仍用"我"，不说"[AI名]来帮你"。
    ```
    如 AI 称呼未取，此条不写。
  - **因为组合 A 没有独立的 AI_INDEX 用于路由，在 AGENT 顶部加一句**："AI 启动时先读本文，再读 NOW.md"
- **行数**：首版控制在 40 行内
- **删除**所有 `<!-- AI注: -->` 注释

### Step A2 · 建 AI_INDEX.md（最小版）

- **模板**：`templates/AI_INDEX.template.md`
- **关键动作**：必读清单只列实际建了的文件——组合 A 只有 AGENT + NOW
- **不列**：PROFILE、heartbeat、memory 等
- **行数**：30 行内

### Step A3 · 建 NOW.md（最小版）

- **模板**：`templates/NOW.template.md`
- **首次填**：让对方口述"现在在做什么 / 接下来打算做什么"
- **大部分字段空着**：写"（首次启动）"即可

### ⏸️ 暂停点 · 让对方扫一眼

> 我已经建了 3 个工作区文件。请扫一眼，告诉我：
> 1) 这个骨架你看着**舒服**吗？（不是问"对不对"——问"舒不舒服"）
> 2) 有没有什么你觉得**多余**或**缺**的东西？
> 现在改还便宜，后面再改要动多处。

**收到反馈后**：根据反馈修订，再进 A4。

### Step A4 · 信号嵌入 → AGENT.md 尾部

遍历**组合 B/C/D/E 中所有未建的元素**，把它们的硬信号写进 AGENT.md 尾部新段。**注意**：如果叠加规则（Q12=A / Q13=A / Q14=A）已建了某条零件，从信号列表中删掉对应行——已建的不需要埋。

```markdown

## 信号埋入 · 未来可能需要升级的零件

> 以下零件当前未建。当触发信号出现时，AI 会主动提醒"该考虑加这个了"。

- [ ] **跨会话连续性（heartbeat + memory + NOW 完整版）**：当你发现每次开 AI 都要重新解释项目背景时 → 升级到组合 B
- [ ] **项目 CHANGELOG**：当你第一次被问到"上次改了啥"而答不上来时 → 建议建
- [ ] **活跃记忆（memory.md）**：当你开始说"以后都按 XXX 来"或纠正同类偏好 ≥3 次 → 建议建
- [ ] **AI 自检速查表（AI_SELF_CHECK）**：当 AGENT 规则 ≥10 条 + 同条规则被口头提醒 ≥2 次 → 建议建
- [ ] **省 token 软提示（AI_SOFT_HINTS）**：当 AI 频繁写"好的，让我来..."等开场白 → 建议建
```

## 你（AI）容易踩的 3 个坑

1. **想替对方做决定**：对方答"你看呢"时，**给选项 + 推荐 + 理由**，不要直接拍
2. **想塞所有功能**：每一步都问自己"这一步对**这个**人真有用吗？"不要因为原作者有就塞
3. **想一次到位**：每个文件首版都有"控制在 X 行内 / 写骨架不写肉"的指引——遵守它，不要因为"不够完整"就加内容
