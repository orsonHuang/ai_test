## 1. 设计目标

### 1.1 验证范围

| 维度 | 目标 |
|------|------|
| **核心循环** | 弱点发现 → BP积攒 → 破盾爆发 → 窗口输出，四阶段策略闭环 |

## 2. 核心战斗机制

### 2.1 Break / Shield 护盾击破系统

**规则**：

- 每个敌人拥有护盾值（3\~5），显示在敌人卡片上
- **只有用弱点属性攻击**才能削减护盾。每次弱点攻击减 1 层，Boost 每级额外 +1 层
- 非弱点属性攻击**不削减护盾**
- 护盾归零 → 敌人进入 **Break 状态**：眩晕、无法行动、受到伤害 ×1.5
- Break 持续 **2 回合**（当前回合 + 下一回合）
- Break 结束后，护盾重置为最大值，可再次被击破

**设计意图**：
> 创造「计划→执行→回报」的策略节奏。玩家需要观察弱点、合理分配 BP 和技能资源，在最佳时机触发 Break，而非无脑输出。

### 2.2 BP Boost 增幅系统

**规则**：

- 每个角色拥有 BP 槽，上限 **5 点**
- 每回合开始时 **自动 +1 BP**（初始 2 点）
- 执行行动时可选择消耗 1\~3 BP：
  - **Boost 攻击**：伤害 ×(1 + 0.3×Lv)，额外削盾
  - **Boost 技能**：伤害 ×(1 + 0.25×Lv)，额外削盾
  - **Boost 防御**：额外获得 2 回合防御 UP
- 消耗 0 BP = 不增幅（普通行动）

**设计意图**：
> 创造「攒资源 vs 立即使用」的张力。BP 是玩家唯一可跨回合积累的战略资源，积攒到 3 BP 打爆发 vs 提前破盾创造输出窗口，是核心决策点。

### 2.3 弱点发现系统

**规则**：

- 每个敌人有 3\~4 个弱点属性（从 10 种属性中选取）
- 初始全部隐藏，显示为 `?`
- 用对应属性击中后**永久揭示**该弱点
- 已发现的弱点用金色标签标记
- 角色攻击属性：武器属性（剑/匕首/杖）+ 技能自带属性（火/冰/雷）

**属性表**：

| 类别 | 属性 | 主要来源 |
|------|------|----------|
| 物理 | 剑 (sword) | 剑士普攻·技能 |
| 物理 | 匕首 (dagger) | 盗贼普攻·技能 |
| 物理 | 杖 (staff) | 学者普攻 |
| 元素 | 火 (fire) | 学者·火焰术 |
| 元素 | 冰 (ice) | 学者·冰霜术 |
| 元素 | 雷 (lightning) | 学者·雷电术 |

### 2.4 行动顺序

- 每回合开始时，根据 **速度 (SPD) + 随机波动 (±10)** 计算行动顺序
- 顶部时间轴从左到右显示本回合行动顺序
- 我方角色和敌方混合排列
- 当前行动者高亮放大，已行动者半透明

**设计意图**：
> 速度差异 + 随机波动创造微观不确定性，使玩家无法完全预测行动序列，增加临场决策的紧张感。

## 4. 敌方设计

### 4.1 Boss — 远古魔像

| 属性 | 值 |
|------|-----|
| HP | 700 |
| ATK | 48 |
| DEF | 22 |
| SPD | 40 |
| 护盾 | 5 |
| 弱点 | 剑 / 火 / 雷 / 匕首 |

**AI 行为**（每回合随机）：

| 权重 | 技能 | 效果 |
|------|------|------|
| 35% | 重击 | 单体 ×1.3 伤害 |
| 30% | 地震 | 全体 ×0.7 伤害 |
| 35% | 硬化 | 自身防御 UP (2回合) |

**设计意图**：
> 5 层护盾需要玩家配合多角色弱点攻击才能击破。防御高、伤害高，破盾窗口是唯一的安全输出期。硬化技能考验玩家是否留有破盾资源——如果在它防御 UP 时破盾，爆发效率减半。

### 4.2 杂兵 A — 岩石傀儡

| 属性 | 值 |
|------|-----|
| HP | 260 |
| ATK | 28 |
| DEF | 24 |
| SPD | 35 |
| 护盾 | 2 |
| 弱点 | 火 / 剑 / 冰 |

**AI**：70% 撞击 / 30% 碎石（均为单体 ×0.9）

**设计意图**：高防御杂兵，与 Boss 共享剑弱点，学者可同时覆盖火/冰弱点。2 层护盾易破，适合盗贼快速处理。

### 4.3 杂兵 B — 水晶魔灵

| 属性 | 值 |
|------|-----|
| HP | 220 |
| ATK | 32 |
| DEF | 14 |
| SPD | 75 |
| 护盾 | 3 |
| 弱点 | 雷 / 匕首 / 冰 |

**AI**：70% 晶化射线（单体 ×1.0）/ 30% 治愈波动（回复 Boss ~40 HP）

**设计意图**：高速治疗单位。必须优先处理或在我方回合打断。速度快，常在我方行动前抢先治疗 Boss，创造「先杀治疗还是先破 Boss 盾」的战略抉择。

## 5. 数值公式

### 5.1 伤害计算

```
基础伤害 = ATK × 技能威力 × 攻击buff修正
破防修正 = 目标Break中 ? 1.5 : 1.0
最终伤害 = max(1, 基础伤害 × 破防修正 - DEF × 0.3)
最终伤害 = 最终伤害 × 随机因子(0.9~1.1 隐含在计算精度中)
```

### 5.2 Boost 修正

| Boost Lv | 攻击伤害 | 技能伤害 |
|----------|----------|----------|
| 0 (无) | ×1.0 | ×1.0 |
| 1 | ×1.3 | ×1.25 |
| 2 | ×1.6 | ×1.5 |
| 3 | ×1.9 | ×1.75 |

### 5.3 护盾削减

| 条件 | 削减量 |
|------|--------|
| 弱点命中 | 1 |
| 弱点命中 + Boost Lv.N | 1 + N |
| 破甲击 | 1 + N + 2 (额外) |
| 非弱点 | 0 |

### 5.4 暴击

- 暗袭基础暴击率 30%，每级 Boost +10%
- 暴击伤害 ×1.5

### 5.5 Buff/Debuff 修正

| Buff | 效果 |
|------|------|
| defUp | 防御 ×1.5 |
| atkUp | 攻击 ×1.4 |
| 防御姿态 | 防御 ×1.8 (仅当回合) |
| 破防加成 | 受伤 ×1.5 |

| Debuff | 效果 |
|--------|------|
| atkDown | 攻击 ×0.6 (敌方) |
| accDown | 命中率 70% (敌方) |

## 6. 战斗流程

```
回合开始
  → 移除死亡单位
  → 所有单位 Buff 倒计时 -1
  → Break 倒计时 -1（Break 中敌人恢复检查）
  → 敌方 AI 决定本回合行动意图（显示在敌人卡片上）
  → 计算行动顺序（SPD + 随机波动，降序排列）
  → 按顺序执行：
      【我方角色】
        → 玩家选择：Boost(0~3) → 攻击/技能/防御
        → 攻击/技能：选择目标 → 执行伤害、削盾、施放效果
        → 防御：立即生效
      【敌方】
        → AI 执行预定行动
        → 受 Debuff 影响（攻击降低/命中率惩罚）
      → 检查战斗结束条件
回合结束
  → 所有存活我方角色 BP +1
  → 回合数 +1
```

### 结束条件

- **胜利**：所有敌人 HP = 0
- **失败**：所有我方角色 HP = 0

## 9. 已知限制 & 后续方向

### 当前版本限制
- 单场预设战斗，敌人配置固定
- 无角色养成/装备/等级成长系统
- 无音效/音乐（仅 UI 点击反馈音）
- 战斗日志不持久化
- 无存档/回放功能

### 可扩展方向（P1）
- 弱点完全揭示后的 UI 优化（不显示 ? 占位）
- Break 期间敌人显示「BREAK」水印 + 动画
- 角色死亡后的站位/动画处理
- 更多敌人技能类型（DoT、驱散、召唤等）
- 道具系统（回复药、BP 药等）
- 多场遭遇战 + 战斗间过渡
