## 一、功能概述

### 1.1 原型名称

即时三消RPG（7×6 棋盘 · 即时制 · 5vN 对战 · 3 波次 · PvE）

### 1.2 核心体验

玩家在实时流逝的战斗中操作下方 7×6 棋盘，通过消除方块驱动上方 5 名英雄攻击敌人。时间压力下，玩家必须快速决策——优先消哪个颜色（决定哪个英雄攻击）、是攒能量出绝招还是持续输出、合成物是直接触发还是留着做连锁。战斗共 3 波，每波敌人更强更多，英雄不回复，必须在策略和手速间找到平衡。

### 1.3 验证目标

| 项目 | 内容 |
|------|------|
| **核心假设** | 即时制三消+RPG 能在时间压力下产生有意义的策略选择，而非沦为纯手速竞赛 |
| **验证判据** | 试玩过程中，玩家在"优先消哪个颜色""攒能量还是直接攻击""何时使用合成物"等决策上有意识的分化行为；至少能自然形成"稳扎稳打"和"赌一波绝招"两种不同打法 |
| **不验证** | 美术风格、音效表现、联网对战、长期养成系统、英雄升级/装备 |

## 二、核心设计

### 2.1 场景/界面规格

- **类型**：上半区 RPG 对战 + 下半区三消棋盘
- **布局**：左侧 200px 调试面板，右侧主区域
- **战斗区**：上方 5 张敌人卡片，下方 5 张英雄卡片（含 HP 条、能量条、状态效果图标）
- **棋盘**：7 列 × 6 行，单元格 60px，4 种颜色（蓝红绿黄）+ 合成物 + 绝招方块
- **调试面板**：实时状态显示、4 个参数滑块、一键重置按钮、可滚动日志区

### 2.2 时序/流程结构

```
idle → playing（第 1 波）
         │
         ▼（敌人全灭）
   wave_transition（2.5s 提示）
         │
         ▼
   playing（第 2 波）→ ... → playing（第 3 波）
         │                        │
         ▼                        ▼
      lose（全灭）            win（通关）
```

### 2.3 元素/实体/组件设计

#### 英雄（5 名）

| 名称 | 职业 | 颜色 | 双色 | ATK | DEF | HP | 能量上限 | 特殊能力 |
|------|------|------|------|-----|-----|-----|----------|----------|
| 泡泡 | 肉盾 | 蓝 | - | 80 | 65 | 550 | 100 | 重击：防御↑；绝招：护盾 |
| 疾风 | 攻击 | 红 | - | 120 | 30 | 360 | 100 | 重击：伤害+流血；绝招：高伤+流血 |
| 针筒天使 | 治疗 | 绿 | - | 65 | 40 | 400 | 100 | 重击：单疗；绝招：群疗+轻伤 |
| 斯巴克 | 技术 | 黄 | - | 95 | 40 | 420 | 100 | 重击：伤害+强化方块；绝招：2敌伤+眩晕+清黄方块 |
| 雷霆王子 | 支援 | 蓝 | 绿 | 85 | 45 | 440 | 100 | 重击：队友+能；绝招：队友+能+驱散 |

#### 敌人

| 波次 | 数量 | ATK | DEF | HP | 技能 |
|------|------|-----|-----|-----|------|
| 1 | 3 | 70 | 20 | 180 | 小伤害（1敌）+ 大伤害（3敌），轮流 |
| 2 | 4 | 90 | 30 | 280 | 同上 |
| 3 | 5 | 110 | 40 | 400 | 同上 |

#### 棋盘元素

| 类型 | 说明 |
|------|------|
| 普通方块 | 4 色（蓝/红/绿/黄），消除触发对应颜色英雄攻击 |
| 强化方块 | 斯巴克重击生成，消除提供 2 倍能量 |
| 横火箭 | 竖直 4 连消除生成，点击消除一行 |
| 竖火箭 | 水平 4 连消除生成，点击消除一列 |
| 炸弹 | 5 连消除生成，点击消除周围 12 格 |
| 黑洞 | 6+ 连消除生成，记录颜色，点击消除同色全方块 |
| 绝招方块 | 英雄能量满后生成，点击触发绝招 |

### 2.4 核心规则

```
规则 1：手动 3 消 → 对应颜色英雄普攻 + 积攒能量（每方块 1 点 × 能量系数）
规则 2：手动 4 消 → 对应颜色英雄重击 + 积攒能量 + 生成火箭
规则 3：手动 5 消 → 对应颜色英雄重击 + 积攒能量 + 生成炸弹
规则 4：手动 6+ 消 → 对应颜色英雄重击 + 积攒能量 + 生成黑洞
规则 5：链式消除（合成物触发/掉落匹配）→ 触发攻击和能量，不生成合成物；攻击等级同手动（3消=普攻，4+消=重击）
规则 6：能量满 100 → 棋盘生成绝招方块
规则 7：双色英雄享受两个颜色的消除收益
规则 8：合成物间可拖拽交互（火箭+火箭=十字；火箭+炸弹=三竖三横；炸弹+炸弹=范围消除）
规则 9：敌人每 4.5 秒交替释放小伤害和大伤害技能
规则 10：波次过渡不回复 HP，不重置棋盘
边界条件：已死亡单位不能被攻击或治疗
特殊情况：流血的初始触发伤算在施放时的伤害中，后续每 3 秒跳一次
```

### 2.5 自动化行为规则

- 链式消除与手动消除使用相同的攻击等级判定（3 消=普攻，4+ 消=重击）
- 合成物间交互完整覆盖（火箭+火箭、火箭+炸弹、炸弹+炸弹）
- 英雄攻击目标：随机选择存活敌人
- 治疗目标：HP 百分比最低的队友
- 所有攻击/治疗/增益均有直线弹道动效（飞弹从来源卡片飞向目标卡片）+ 受击闪白 + 浮动数字

### 2.6 完成/结束条件

| 结果 | 条件 |
|------|------|
| 胜利 | 第 3 波全部敌人被击败 |
| 失败 | 5 名英雄全部阵亡 |

### 2.7 交互流程

```
打开页面 → 自动开始战斗
  → 观察棋盘（7×6，4 色方块 + 合成物/绝招方块）
  → 点击方块 A 选中（高亮） → 点击相邻方块 B 完成交换
    → 系统自动：判定匹配 → 消除 → 英雄攻击 → 棋盘重力 → 新方块填充 → 链式消除
    → 能量满时自动生成绝招方块（脉冲动画）
  → 点击绝招方块释放绝招
  → 点击合成物（火箭/炸弹/黑洞）直接触发消除效果
  → 拖拽合成物到另一个合成物进行连锁交互
  → 敌人在计时器到达时自动攻击
  → 波次清空后 2.5s 过渡 → 下一波
  → 左侧调试面板实时更新状态 + 可热调参数
```

## 三、配置项清单

### CONFIG 参数

| 参数名 | 默认值 | 可调范围 | 生效方式 | 说明 |
|--------|--------|----------|----------|------|
| `HERO_DAMAGE_COEFF` | 1.0 | 0.1 ~ 3.0 | 即时 | 英雄所有伤害倍率 |
| `HERO_ENERGY_COEFF` | 1.0 | 0.1 ~ 3.0 | 即时 | 英雄能量获取倍率 |
| `ENEMY_ATTACK_CD` | 4.5s | 1.0 ~ 10.0 | 即时 | 敌方攻击间隔 |
| `GAME_SPEED` | 1.0x | 0.5 ~ 3.0 | 即时 | 全局速度（含敌人攻击频率） |

### 伤害公式

```javascript
damage = max(ATK * ATK / (ATK + DEF * 2), 1)
// 最终伤害 = damage × HERO_DAMAGE_COEFF（英雄） 或 damage × ENEMY_DAMAGE_COEFF（敌方）
```

**数值示例**：

| 攻击方 ATK | 防御方 DEF | 基础伤害 | 说明 |
|-----------|-----------|---------|------|
| 120（疾风） | 20（波1敌） | 90 | 高攻对低防 |
| 120（疾风） | 40（波3敌） | 72 | 高攻对中防 |
| 65（针筒） | 40（波3敌） | 29 | 低攻对中防 |
| 70（波1敌） | 45（雷霆） | 30 | 敌攻对中防 |

### 能量阈值

```javascript
能量获取 = 消除方块数 × 能量系数（强化方块 ×2）
能量上限 = 100
绝招消耗 = 100（释放后归零）
```

## 四、验收清单

### 4.1 基础交互

- [ ] 点击棋盘方块 → 方块高亮（放大+白色边框）
- [ ] 点击相邻方块 → 方块交换并判定消除
- [ ] 点击已选中的方块 → 取消选中
- [ ] 点击不相邻方块 → 不会发生交换
- [ ] 点击绝招方块（脉冲动画方块）→ 绝招释放
- [ ] 点击合成物（火箭/炸弹/黑洞）→ 对应消除效果触发
- [ ] 拖拽合成物到另一个合成物 → 连锁效果触发
- [ ] 一键重置按钮 → 游戏回到初始状态

### 4.2 核心逻辑

- [ ] 3 消 → 只有普攻动画/日志
- [ ] 4 消 → 有重击 + 合成物出现在棋盘上
- [ ] 5 消 → 炸弹生成
- [ ] 6+ 消 → 黑洞生成
- [ ] 消除蓝色方块 → 泡泡和雷霆王子都攻击/积能
- [ ] 消除绿色方块 → 针筒天使和雷霆王子都攻击/积能
- [ ] 能量攒满 → 绝招方块出现在棋盘上
- [ ] 疾风重击 → 敌人出现流血图标，每 3 秒跳伤害
- [ ] 针筒天使重击 → 最低血量英雄回复
- [ ] 泡泡重击 → 自身防御提升
- [ ] 斯巴克重击 → 棋盘出现强化方块（带 ✦ 标记）
- [ ] 雷霆王子重击 → 队友获得能量
- [ ] 泡泡绝招 → 队友出现护盾
- [ ] 斯巴克绝招 → 全部黄色方块消失
- [ ] 敌人每 4.5 秒攻击一次
- [ ] 波次 1 清空 → 2.5s 后进入波次 2（敌人变多）
- [ ] 波次过渡不回复英雄 HP
- [ ] 波次过渡不重置棋盘
- [ ] 全英雄阵亡 → 显示"失败"
- [ ] 第 3 波全清 → 显示"胜利"

### 4.3 调试面板

- [ ] 左侧面板实时显示：游戏状态、当前波次、游戏时间、英雄存活数、敌人生存数
- [ ] 拖动"伤害系数"滑块 → 后续伤害即时变化
- [ ] 拖动"能量系数"滑块 → 能量获取即时变化
- [ ] 拖动"敌方攻击 CD"滑块 → 敌人攻击间隔变化
- [ ] 拖动"攻击速度"滑块 → 游戏整体节奏变化
- [ ] 日志区持续记录关键事件（有时间戳）

### 4.4 边界条件

- [ ] 当英雄全部阵亡后，棋盘仍可操作但不触发攻击
- [ ] 敌人全部被击败后，不再发动攻击
- [ ] 狂暴点击（快速连续交换）不会导致状态异常
- [ ] 棋盘无匹配的交换会被自动换回
- [ ] 链式消除不产生新的合成物（只触发攻击）
