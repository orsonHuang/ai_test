## 二、玩法概述

### 2.1 一句话描述

四人小队回合制 RPG 战斗，通过 BP 增幅系统、弱点破防机制和技能搭配，在 3 波敌人中寻找最优战术路径。

### 2.2 核心循环

```
轮到角色 → 选择行动(攻击/技能/防御) → 选择 BP 消耗 → 选择目标 → 执行(伤害/治疗/破防) → 行动条下一位 → ...
破坏循环的扳机：敌人破防(爆发窗口)、队友阵亡(压力上升)、BP 满(决策峰值)
```

### 2.3 玩法分类

- **主类型**：[✓] 战斗/对抗  [ ] 活动/小玩法  [ ] 叙事/剧情  [ ] 经济/资源流  [ ] 构筑/卡组
- **对战模式**（战斗类）：[ ] PvP  [✓] PvE  [ ] 单机
- **参与时长**（单局/单次）：2~5 分钟（3 波战斗）

### 3.2 差异化亮点

> [待策划填写]

- **BP + 破防双系统耦合**：歧路旅人的两个核心系统被抽离为独立可调参数，可以通过调试面板独立验证每个系统对策略深度的影响
- **技能系统热调友好**：每个技能的 SP 消耗和倍率在代码中结构化定义，修改一个技能只需改一个对象

### 3.3 主动放弃了什么

| 排除项 | 排除原因 |
|--------|---------|
| 养成/成长系统 | 本次只验证"单场战斗内"的策略深度，不验证跨场资源管理 |
| 移动/网格系统 | 参照歧路旅人的队列对排模式，聚焦技能选择而非位置博弈 |
| 多角色切换/替补 | 4 人固定上场，减少 UI 复杂度，聚焦技能组合 |
| 装备/道具系统 | 原型阶段引入装备会稀释 BP 和破防的核心体验 |
| 美术立绘/音效 | 原型目标是快速验证规则，反馈通过 CSS 动画和数字特效实现 |

## 四、数值平衡目标

### 4.1 平衡判据

| 平衡目标 | 验收标准 | 当前状态 |
|---------|---------|---------|
| 多路线可行 | 试玩 5 局，至少出现 3 种不同打法（如：速攻破防 / 攒 BP 爆发 / 控场消耗） | [ ] 未测 |
| 无角色冗余 | 4 个角色在试玩中均有"核心贡献时刻"（即被选中为 MVP 操作者） | [ ] 未测 |
| 3 波递增强度合理 | 第 1 波可在 3~5 回合内通过，第 3 波需要策略规划才能通过 | [ ] 未测 |

### 4.2 关键数值的设计理由

| 参数名 | 默认值 | 设计理由 |
|--------|--------|---------|
| BP_RATE = 1 | 1 | 每回合 1 BP 意味着 5 回合攒满，恰好对应 3 波战斗的自然节奏——前中期需要决策"花还是攒"，后期 BP 满后爆发窗口集中 |
| BREAK_DMG_BONUS = 1.5 | 1.5 | 1.5× 既能让破防有明显收益（鼓励弱点策略），又不至于让非破防输出完全失去意义。太高（2×+）会导致"不破防不打" |
| SP_REGEN = 5 | 5 | 低 SP 角色（剑士 60）约 10 回合自然回满，高 SP 角色（学者 150）约 30 回合——制造"SP 资源管理"的中期紧张感，而非无限使用 |

## 五、类型专项 — 战斗/对抗类

### AI 设计意图

> [待策划填写]

当前 AI 行为目标：提供"有威胁感的对手"，而非最优化对手。具体表现：
- 精英在蓄力→释放的周期中制造"拆弹"紧迫感
- 法师的 DEF 增益使物理输出角色需要依赖破防窗口
- 纯 HP 最低优先攻击（P1 已知问题：威胁评估未实现），使战斗结果偏向可预测

### 胜负条件的设计理由

> [待策划填写]

选择"消灭全部敌人"作为胜利条件，而非点数/时间——因为本原型的核心验证目标是"策略多样性"，消灭条件最能暴露是否存在"万能路线"。
