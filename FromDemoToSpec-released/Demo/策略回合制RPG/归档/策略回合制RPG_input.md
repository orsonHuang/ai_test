## 二、必填区

### A. 验证目标

- **要验证的核心假设是什么？**
  回合制 RPG 小队战斗（BP 增幅 + 破防机制 + 弱点系统）能否产生丰富的策略空间，避免“最优解”——即在 4 角色 6 技能的组合下，不同打法路线都有赢面。

- **验证成功的判据是什么？**
  试玩 5 局，至少出现 3 种不同的打法思路（如：主破防速攻 / 攒 BP 爆发 / 控场消耗等）。

- **不需要验证什么？**
  不需要验证：美术风格/立绘、音效/BGM、长期养成/成长系统、多场战斗间的进度系统、移动端适配、存档/读档。

### B. 核心交互

- **用户做什么操作？**
  轮到己方角色时：选择行动类型（攻击 / 技能 / 防御）→ 选择 BP 增幅档位（0~3 BP）→ 选择目标（敌方或友方）→ 执行行动并观看动画反馈。

- **系统如何响应？**
  播放技能动画 → 弹出伤害/治疗数字 → 检查破防（弱点命中扣盾，盾归零触发眩晕） → 更新 HP/SP/BP → 检查胜负 → 切换行动者 → 若为敌方回合，AI 自动决策。

- **交互循环（用箭头表示）**
  ```
  行动条决定顺序 → 轮到角色 → 选行动+BP+目标 → 执行 → 动画播放 → 状态更新 → 行动条下一位 → ...
  直到：敌方全灭（进入下一波/胜利）或 我方全灭（失败）
  ```

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Character | id, name, job, hp, maxHp, sp, maxSp, atk, def, spd, bp, attackType, skills[], buffs[], isGuarding, isDead | 属于 party 数组；可对 enemy 造成伤害或对 ally 施加效果 |
| Enemy | id, name, hp, maxHp, atk, def, spd, weaknesses[], shield, maxShield, isBroken, chargeCountdown | 属于 enemies 数组；是角色的攻击目标 |
| Skill | name, type, target, multiplier, spCost, bpCost, desc | 属于 Character；决定伤害类型和效果 |
| ActionQueue | 有序列表 [{type: 'party'/'enemy', index: N}] | 由所有存活单位的 spd 排序决定 |
| BattleState | phase, currentWave, totalWaves, actionQueue, currentActor, turnCount, selectedSkill, bpToSpend, selectedTarget | 全局唯一 |

### D. 状态与流程

- **系统有哪些状态？**
  `idle`（等待开始）→ `wave_intro`（波次提示）→ `player_turn`（等待玩家操作）→ `enemy_turn`（AI 自动行动）→ `animating`（动画播放中）→ `wave_clear`（波次清空）→ `victory`（全部通关）→ `defeat`（队伍全灭）

- **状态之间如何切换？**
  ```
  idle           → （点击"开始"按钮）                     → wave_intro
  wave_intro     → （延迟 1.5s）                          → 构建行动队 → player_turn / enemy_turn
  player_turn    → （玩家完成操作→行动执行完毕）            → 检查波次 → player_turn / enemy_turn / wave_clear / defeat
  enemy_turn     → （AI 行动执行完毕，延迟）                 → 检查波次 → player_turn / enemy_turn / wave_clear / defeat
  wave_clear     → （延迟 1s）                             → 下一波 wave_intro / victory
  victory        → （展示胜利画面）                         → 可点击重置 → idle
  defeat         → （展示失败画面）                         → 可点击重置 → idle
  ```

- **初始状态**：`idle`，4 名角色满状态，当前波次 = 1。
- **结束状态**：`victory`（3 波全通）或 `defeat`（全灭）。

### E. 核心规则

```
- 规则1：行动顺序由 SPD 决定，显示在顶部行动条
- 规则2：每回合角色可选择 攻击 / 技能 / 防御，可选消耗 BP（0~3）增幅
- 规则3：BP 系统：每个角色回合开始时 BP += BP_RATE（上限 5）。普攻消耗 BP = 多打 N 刀（每刀减 1 盾）；技能消耗 BP = 伤害增幅（1BP ×1.5 / 2BP ×2.0 / 3BP ×3.0）
- 规则4：破防机制：敌人有 N 层盾，每层对应弱点类型。命中弱点扣 1 盾，非弱点不扣盾。盾归零 → 敌人进入"破防"状态 1 回合（无法行动 + 受伤倍率上升）
- 规则5：伤害公式：damage = max(1, floor(ATK × 技能倍率 × BP倍率 × 破防加成 - DEF × 0.3)) × DAMAGE_MULT
- 规则6：SP 消耗：技能消耗 SP，普攻不消耗。每回合开始时恢复 SP_REGEN 点 SP
- 规则7：防御：角色进入防御姿态，本回合受伤 -50%，BP 仍然正常增长
- 规则8：角色 HP ≤ 0 时死亡，从行动队中移除
- 规则9：3 波敌人，每波清空后进入下一波
- 边界条件（必须不能发生的事）：
    - 不允许 SP 不足时使用技能
    - 不允许 BP 不足时进行对应档位增幅
    - 不允许选择已死亡的目标
    - 不允许在非 player_turn 状态下响应点击
- 特殊情况处理：
    - 破防状态持续到敌人下一次回合开始（即破防后到下次轮到该敌人行动为止）
    - 敌人破防恢复后盾值重置为满
    - 伤害最低为 1（不能为 0 或负数）
    - 精英蓄力技：预告 2 回合，到期释放高伤，被破防则取消蓄力
```

### F. UI 骨架

- **整体布局**：
  顶部行动条（横向滚动）| 中部：左侧 敌人队列 + 中央战斗区（动画/数字/特效） + 右侧 己方队列 | 底部：操作面板 + 状态栏

- **关键视觉元素的位置**：
  - 行动条：屏幕顶部横向排列，当前行动者高亮
  - 敌人：左侧竖向排列，显示名称/HP/盾值/弱点
  - 己方：右侧竖向排列，显示名称/HP 条/SP 条/BP 圆点
  - 伤害数字：从目标头顶飘出，向上浮动消失
  - 操作面板：底部居中，行动选择 → BP 选择 → 目标选择（分步式）
  - 破防状态：敌人身上显示"BREAK"标记 + 震动动画

- **信息展示优先级**：
  1. 当前行动者是谁（行动条高亮 + 操作面板提示）
  2. 己方 HP/SP/BP 状态
  3. 敌人 HP/盾值/弱点
  4. 波次信息
  5. 战斗日志

### G. 可调参数（调试面板数据源）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| DAMAGE_MULT | 1.0 | 0.5 ~ 3.0 | 全局伤害倍率 |
| ENEMY_HP_MULT | 1.0 | 0.5 ~ 3.0 | 敌人血量倍率 |
| BP_RATE | 1 | 1 ~ 3 | 每回合 BP 增长量 |

## 三、选填区（按勾选模块）

### [✓] 战斗/对抗类

#### 对战模式
PvE（玩家 vs AI），AI 控制敌方。

#### 回合结构

每回合的步骤（按顺序）：
```
1. 行动条决定当前行动者
2. 若为玩家角色：展示操作面板（攻击/技能/防御）
3. 若为敌人：AI 自动决策并执行
4. 行动完毕 → 检查波次状态 → 切换下一位
5. 全部行动完毕？重新按 SPD 排序，新一轮
```

#### 战场规格
- 维度：无网格，队列对排型（敌左我右）
- 尺寸：不适用（无网格）
- 特殊地形：无

#### 单位/角色模板

| 名称 | 类型 | HP | SP | ATK | DEF | SPD | 弱点覆盖 | 技能 |
|------|------|-----|-----|-----|-----|-----|---------|------|
| 剑士 | 斩物理 | 800 | 60 | 120 | 80 | 70 | 斩 | 横斩（斩全体 0.7×）、蓄力斩（斩单体 2.5×，固定耗 2BP） |
| 学者 | 元素 | 500 | 150 | 140 | 40 | 60 | 火/冰 | 火球术（火单体 1.8×）、冰霜风暴（冰全体 1.2×） |
| 神官 | 治疗/光 | 650 | 120 | 80 | 60 | 80 | 打/光 | 治愈术（回复 ATK×0.8 HP）、圣光（光单体 1.6× + 自回） |
| 舞娘 | 突/辅助 | 550 | 100 | 100 | 50 | 110 | 突 | 月光之舞（全体 ATK↑ 2回合）、暗影步（突单体 1.4× + 拉队友行动条） |

#### 敌人模板

| 名称 | HP | ATK | DEF | SPD | 弱点 | 盾值 | 特殊行为 |
|------|-----|-----|-----|-----|------|------|---------|
| 狼 | 300 | 60 | 30 | 50 | 斩 | 2 | 纯输出 |
| 法师 | 400 | 80 | 40 | 70 | 突/光 | 3 | 会给自己上 DEF↑ |
| 精英 | 900 | 120 | 70 | 40 | 打/冰 | 4 | 高伤害 + 蓄力技（2回合预警） |

#### 伤害公式
```
baseDamage = ATK × skillMultiplier × bpMultiplier × breakBonus - DEF × 0.3
finalDamage = max(1, floor(baseDamage)) × DAMAGE_MULT

bpMultiplier（技能）: 1.0 (0BP), 1.5 (1BP), 2.0 (2BP), 3.0 (3BP)
bpMultiplier（普攻）: 1.0 + 每 1BP = 额外 1 击
breakBonus = 1.5 (if broken) else 1.0
```

#### AI 行为规则
```
优先级（从高到低）：
1. 若可蓄力技且蓄力完毕 → 释放蓄力技
2. 若可蓄力技且未蓄力 → 开始蓄力（2回合倒计时）
3. 普通法师 → 若自身无 DEF 增益 → 给自己上 DEF↑ buff
4. 攻击：优先选择能破其弱点的角色（即攻击类型匹配该角色弱点的敌人优先打），其次打 HP 最低的角色
5. 精英被破防期间 → 蓄力被取消，恢复后重新蓄力
```

#### 胜负条件
```
胜利：3 波敌人全部消灭
失败：我方 4 名角色全部阵亡
```
