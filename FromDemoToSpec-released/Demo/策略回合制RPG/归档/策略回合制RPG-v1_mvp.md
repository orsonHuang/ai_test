## 一、功能概述

### 1.1 原型名称 / 功能名称

策略回合制RPG（4 人小队 · BP 增幅 · 弱点破防 · 3 波 PvE）

### 1.2 核心体验

玩家操控一支 4 人冒险小队（剑士、学者、神官、舞娘），在回合制战斗中面对 3 波敌人。每回合需要决策"用哪个技能、花多少 BP、打哪个目标"——这是一个多维度取舍：BP 是攒着爆发还是现在花掉？先破防精英还是清小怪？用神官回血还是用圣光输出？

战斗反馈强烈：伤害数字跳动、破防瞬间屏幕震动、BP 满时金光闪烁。击败一波敌人后短暂喘息，随即面对更强阵容。

### 1.3 验证目标

| 项目 | 内容 |
|------|------|
| **核心假设** | BP 增幅 + 弱点破防双系统叠加，能产生多路线策略空间，不会被单一"最优解"统治 |
| **验证判据** | 试玩 5 局，至少出现 3 种不同的打法思路（如：主破防速攻 / 攒 BP 爆发 / 控场消耗） |
| **不验证** | 美术立绘、音效 BGM、长期养成系统、跨场次资源管理、移动端适配、存档读档 |

## 二、核心设计

### 2.1 场景/界面规格

- **类型**：队列对排型回合制战斗（无网格）
- **布局**：
  - 顶部：行动条（横向排列所有单位，按行动顺序）
  - 中部左：敌人队列（竖向排列）
  - 中部右：己方队列（竖向排列）
  - 中部中：战斗特效区（伤害数字、技能动画）
  - 底部：操作面板（行动选择 → BP 选择 → 目标选择） + 状态栏（HP/SP/BP/Buff）
- **特殊元素**：敌人破防时震动动画 + "BREAK" 标记；BP 满时金色光效

### 2.2 时序/流程结构

```
idle → wave_intro → [player_turn ↔ enemy_turn] × N → wave_clear → ... → victory/defeat
```

- `idle`：开始界面，显示"开始战斗"按钮
- `wave_intro`：显示当前波次（1/3），1.5s 后进入战斗
- `player_turn`：轮到己方角色，操作面板激活
- `enemy_turn`：AI 自动决策，短暂延迟后执行
- `wave_clear`：波次完成提示，1s 后进入下一波或胜利
- `victory` / `defeat`：展示结果，可点击重置

### 2.3 元素/实体/组件设计

#### 角色

| 名称 | 职业 | HP | SP | ATK | DEF | SPD | 普攻类型 | 技能 |
|------|------|-----|-----|-----|-----|-----|---------|------|
| 剑士 | 战士 | 800 | 60 | 120 | 80 | 70 | 斩 | 横斩（斩·全体·0.7×·12SP）、蓄力斩（斩·单体·2.5×·18SP·固定耗2BP） |
| 学者 | 法师 | 500 | 150 | 140 | 40 | 60 | 打 | 火球术（火·单体·1.8×·10SP）、冰霜风暴（冰·全体·1.2×·22SP） |
| 神官 | 治疗 | 650 | 120 | 80 | 60 | 80 | 打 | 治愈术（回复 ATK×0.8·8SP）、圣光（光·单体·1.6×·14SP + 自回20%伤害） |
| 舞娘 | 辅助 | 550 | 100 | 100 | 50 | 110 | 突 | 月光之舞（全体 ATK↑30%·2回合·15SP）、暗影步（突·单体·1.4×·10SP + 拉队友行动条前移2位） |

#### 敌人（3 波）

| 波次 | 敌人组合 |
|------|---------|
| Wave 1 | 狼 ×2 + 法师 ×1 |
| Wave 2 | 狼 ×2 + 法师 ×1 + 精英 ×1 |
| Wave 3 | 狼 ×1 + 法师 ×2 + 精英 ×1 |

| 名称 | HP | ATK | DEF | SPD | 弱点 | 盾值 | 特殊行为 |
|------|-----|-----|-----|-----|------|------|---------|
| 狼 | 300 | 60 | 30 | 50 | 斩 | 2 | 纯输出 |
| 法师 | 400 | 80 | 40 | 70 | 突、光 | 3 | 若无 DEF 增益 → 给自己加 DEF↑（+30% DEF，持续 3 回合） |
| 精英 | 900 | 120 | 70 | 40 | 打、冰 | 4 | 每隔 3 回合蓄力，2 回合后释放高伤技（ATK×2.0），破防可打断 |

### 2.4 核心规则

```
规则1 — 行动顺序：所有存活单位按 SPD 排序构成行动队。舞娘「暗影步」可将队友前移。
规则2 — BP 增幅：每个角色回合开始时 BP += BP_RATE（上限 5）。普攻消耗 BP 可多段攻击；技能消耗 BP 可倍率增幅。
规则3 — 弱点破防：敌人弱点的攻击类型命中时，盾值 -1。盾归零 → 进入"破防"状态（眩晕+受伤 1.5×），持续至该敌人下一回合开始。恢复后盾值重置。
规则4 — SP 消耗：普攻 0 SP，技能消耗 8~22 SP。回合开始时恢复 SP_REGEN（默认 5）点。
规则5 — 防御：选择防御后本回合受伤 -50%，BP 正常增长。
规则6 — 伤害计算：damage = max(1, floor(ATK × 倍率 × BP增幅 × 破防加成 - DEF × 0.3)) × DAMAGE_MULT
规则7 — 死亡处理：HP ≤ 0 立即从行动队中移除。
规则8 — 3 波敌人：每波清空后自动进入下一波。
```

**BP 倍率表**：

| 消耗 BP | 普攻效果 | 技能效果 |
|---------|---------|---------|
| 0 | 1 击 | ×1.0 |
| 1 | 2 击 | ×1.5 |
| 2 | 3 击 | ×2.0 |
| 3 | 4 击 | ×3.0 |

**边界条件**：
- 不允许 SP 不足时使用技能
- 不允许 BP 不足时选择对应增幅档位
- 不允许选择已死亡目标
- 不允许在非 player_turn 状态响应操作
- 伤害最低为 1（不能为 0 或负数）

**特殊情况处理**：
- 破防恢复：盾值重置为满，破防状态仅持续 1 回合
- 精英蓄力中断：被破防时取消蓄力，恢复后重新开始蓄力周期
- 拉行动条：最多前移 2 位，不会移出队首

### 2.5 自动化行为规则

敌人 AI 优先级（从高到低）：

1. **若蓄力技已准备完毕** → 释放蓄力技（对 HP 最低角色）
2. **若可蓄力技但未开始蓄力** → 开始蓄力（3 回合倒计时）【仅精英】
3. **若为法师且自身无 DEF 增益** → 给自己加 DEF↑ buff
4. **攻击目标选择**：攻击 HP 最低的角色（P1 已知问题：`findTargetByThreat` 威胁评估未实现，暂无条件优先匹配弱点角色）
5. **若被破防** → 跳过回合

信念度标注：强信念（基于主流 RPG AI 设计模式 + 歧路旅人参考）。

### 2.6 完成/结束条件

| 结果 | 条件 |
|------|------|
| 胜利 | 3 波敌人全部消灭 |
| 失败 | 己方 4 名角色全部阵亡 |

### 2.7 交互流程

```
打开页面 → 看到 4 角色状态 + "开始战斗"按钮
  → 点击开始 → 波次提示 → 行动条展开
  → 轮到角色（操作面板激活）：
     → [攻击] → 选 BP 档位 → 选目标 → 伤害动画 + 数字弹出
     → [技能] → 选技能 → 选 BP 档位 → 选目标 → 技能动画 + 数字弹出
     → [防御] → 防御标记显示 → 回合结束
  → 敌人 AI 自动行动 → 伤害动画
  → 波次清空 → "Wave Clear!"
  → 3 波全清 → "Victory!" → 可重置
  → 全灭 → "Defeat" → 可重置
```

## 三、配置项清单

| 参数名 | 默认值 | 可调范围 | 生效方式 | 说明 |
|--------|--------|----------|----------|------|
| `DAMAGE_MULT` | 1.0 | 0.5 ~ 3.0 | 即时 | 全局伤害倍率 |
| `ENEMY_HP_MULT` | 1.0 | 0.5 ~ 3.0 | 重置后 | 敌人最大 HP 倍率 |
| `BP_RATE` | 1 | 1 ~ 3 | 即时 | 每回合 BP 增长量 |
| `SHIELD_MULT` | 1.0 | 0.5 ~ 2.0 | 重置后 | 敌人初始盾值倍率 |
| `SP_REGEN` | 5 | 0 ~ 20 | 即时 | 每回合 SP 恢复量 |
| `BREAK_DMG_BONUS` | 1.5 | 1.0 ~ 3.0 | 即时 | 破防后受伤倍率加成 |

### 伤害公式详解

```
baseDamage = ATK × skillMultiplier × bpMultiplier × breakBonus - DEF × 0.3
finalDamage = max(1, floor(baseDamage)) × DAMAGE_MULT
```

### 数值示例

| 场景 | ATK | 技能倍率 | BP | 破防 | DEF | 计算过程 | 伤害 |
|------|-----|---------|-----|------|-----|---------|------|
| 剑士普攻狼 | 120 | 1.0 | 0 | 否 | 30 | 120×1.0×1.0 - 30×0.3 = 111 | 111 |
| 剑士蓄力斩破防精英 | 120 | 2.5 | 2BP(固定) | 是(1.5×) | 70 | 120×2.5×1.0×1.5 - 70×0.3 = 429 | 429 |
| 学者火球术破防法师 | 140 | 1.8 | 2BP(2.0×) | 是(1.5×) | 40 | 140×1.8×2.0×1.5 - 40×0.3 = 744 | 744 |
| 舞娘暗影步狼(1BP) | 100 | 1.4 | 1BP(1.5×) | 否 | 30 | 100×1.4×1.5 - 30×0.3 = 201 | 201 |

## 四、验收清单

### 4.1 基础交互

- [ ] 页面加载后，能看到 4 名角色信息（名称、HP、SP、BP）
- [ ] 点击"开始战斗"，波次提示显示后进入战斗
- [ ] 玩家回合时，操作面板可交互；敌人回合时，操作面板禁用
- [ ] 可选择"攻击"，选 BP 档位（0~当前 BP 上限），选目标，执行成功
- [ ] 可选择"技能"，选技能 → 选 BP → 选目标，执行成功
- [ ] 可选择"防御"，执行成功（防御标记可见）
- [ ] 战斗结束后（胜利/失败），可点击重置回到初始状态

### 4.2 核心逻辑

- [ ] BP 在角色回合开始时正确 +1（或 BP_RATE 设定值）
- [ ] BP 上限为 5，不会溢出
- [ ] 普攻 1 BP 打出 2 段伤害动画，每段独立扣盾
- [ ] 技能消耗正确 SP 量，SP 不足时不可用
- [ ] 命中弱点时敌人盾值 -1（可见）
- [ ] 非弱点攻击不扣盾（盾值不变）
- [ ] 盾值归零 → 敌人显示"BREAK"状态，跳过其下回合
- [ ] 破防状态下伤害明显提高（约 1.5 倍）
- [ ] 破防恢复后盾值重置为满
- [ ] 防御状态下受伤约减半
- [ ] 3 波敌人依次出现，波次间有过渡提示
- [ ] 全灭敌人 / 全灭己方时游戏结束并显示结果
- [ ] 精英会蓄力（显示倒计时），到期释放高伤，破防可打断

### 4.3 调试面板

- [ ] 左侧状态面板实时显示：当前回合、当前行动者、波次、各角色 HP/SP/BP
- [ ] 右侧 6 个滑块均可拖动，当前值即时显示
- [ ] DAMAGE_MULT 滑块拖动后，下次攻击伤害立即变化
- [ ] ENEMY_HP_MULT 滑块拖动后，重置游戏后敌人 HP 变化
- [ ] 一键重置按钮可用，点击后回到初始状态（不刷新页面）
- [ ] 日志区记录关键事件（行动、伤害、破防、胜负）

### 4.4 边界条件（用户视角）

- [ ] SP 不足时，对应技能按钮灰色不可点击
- [ ] BP 不足时，对应增幅档位不可选
- [ ] 敌人破防状态下，对该敌人攻击伤害明显提高
- [ ] 已死亡角色不出现在行动队中，不可被选为目标
- [ ] 敌人回合中，玩家操作面板完全禁用
- [ ] 全部角色死亡后，游戏正确判定失败

### 4.5 容错与韧性（开发者视角）

- [ ] 快速连续点击操作按钮不会导致状态错乱
- [ ] 伤害计算不会出现 NaN 或负数
- [ ] 行动队为空（全部死亡）时正确判定失败
- [ ] 破防判定在敌人盾值 = 0 时正确触发且仅触发一次
- [ ] 多次重置后状态完全恢复初始值
