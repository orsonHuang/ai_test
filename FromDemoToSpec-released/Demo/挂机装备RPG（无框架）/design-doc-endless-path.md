# 简介

### 设计目的

1. **打造一款以"极致成长爽感"为核心体验的挂机放置类游戏**，玩家通过不断开宝箱获得随机词条装备，持续提升战力，挑战更高关卡
2. **提供极低决策负担的休闲体验**——核心循环自动化（宝箱自动生产+自动开启），唯一手动决策点为装备对比替换
3. **通过夸张的动效反馈制造爽感**——震屏、大字伤害数字、品质光柱、全屏闪光，让每一次装备提升和战斗击杀都有强烈的正反馈
4. **关卡式数值验证提供明确的成长标尺**——战后统计面板让玩家清晰感知自己变强的过程

### 设计内容

1. **宝箱系统**——核心资源产出入口，自动生产/自动开启/手动加速的三态循环
2. **装备系统**——13槽位×10品质×24词条的随机装备生成与对比替换
3. **装备升级**——金币消耗统一提升全局装备等级，带动全属性水涨船高
4. **关卡战斗**——线性推关的全自动即时战斗，6项战后数据统计
5. **动效反馈**——覆盖开箱/出货/战斗/升级全链路的夸张视觉反馈

### 名词解释（*）

| **名词** | **说明** |
|---|---|
| 战力 | 装备战斗力的简称，由各词条加权求和得出，是装备好坏的一眼判断标准 |
| 词条 | 装备上携带的属性条目，每条包含类型（如攻击力）和数值 |
| 品质 | 装备稀有度等级，从普通到至尊共10档，决定词条数量和数值倍率 |
| 装备等级 | 全局概念，所有已装备/未来获得的装备共享此等级加成，通过消耗金币提升 |
| 宝箱 | 装备产出的唯一来源，自动累积/手动或自动开启 |
| 关卡 | 线性推进的战斗验证场景，每关有固定的敌人属性 |

# 方案概述

## 核心循环

```
宝箱自动累积 → 自动/手动开箱 → 装备对比替换/出售
                                    ↓
                              战力提升 → 挑战关卡 → 战后统计 → 获得金币 → 升级装备等级
                                    ↓
                              战力进一步提升 → 挑战更高关卡 → 循环
```

# 设计决策理由

| 决策点 | 选择了什么 | 排除了什么 | 理由 | 对实现的约束 |
|--------|-----------|-----------|------|-------------|
| 装备产出方式 | 宝箱自动生产+开启 | 杀怪掉落 | 避免"打得快=产出快"的正反馈循环导致数值崩坏；宝箱限速控制产出节奏，手动点击提供"我在加速"的掌控感 | 宝箱上限100个，产速3秒/个不可调 |
| 装备对比决策 | 战力低→自动出售；战力高→弹窗选择 | 每次都弹窗/全自动替换 | 避免垃圾装备打断节奏，又保留好装备的选择仪式感 | 需要战力计算权重合理，确保"战力高=确实更好" |
| 装备等级 | 全局等级统一提升 | 单件装备独立等级 | 避免"好装备但等级低 vs 差装备但等级高"的二难选择，让每次装备替换都是纯粹的品质/词条决策 | 装备等级只影响基础属性，不影响词条 |
| 战斗方式 | 全自动即时战斗 | 回合制/半即时 | 放置类玩家期望"看戏"而非操作，全自动+数字洪水满足爽感 | 需要平衡攻速/吸血等实时属性 |
| 视觉风格 | 极简暗黑UI | 花哨装饰/明亮色调 | 极简背景让夸张动效（大字/光柱/震屏）更有冲击力，反差制造爽感 | 动效颜色需在暗色背景上高对比度 |

# 系统功能

## 1. 宝箱系统

1. **宝箱生产规则**
    1. 每 **3秒** 自动生产 **1个** 宝箱
    2. 宝箱上限 **100个**，到达上限后停止生产
    3. 宝箱数量显示在宝箱图标右下角（红底白字）
    4. 宝箱图标状态：无宝箱时无特效，有宝箱时金色光晕，满仓时强烈脉冲光晕

2. **宝箱开启规则**
    1. **自动开启**：每 **5秒** 自动开启1个（可开关）
    2. **手动开启**：点击宝箱图标立即开启1个
    3. 开启时宝箱图标播放弹跳动画（缩放1.0→1.3→0.9→1.0）
    4. 开出的装备战力低于当前同部位 → 自动出售，获得金币，不弹窗
    5. 开出的装备战力高于当前同部位（或无装备） → **暂停所有宝箱生产和开启**，弹出对比面板
    6. 对比面板关闭后，**自动开启计时器归零重新计时**

3. **装备对比面板**
    1. 左栏显示当前装备（品质色名称+词条+战力）
    2. 右栏显示新装备（品质色名称+词条+战力+增减箭头）
    3. 右侧边框随品质变化颜色
    4. 两个按钮：【装备】（金色）和【出售（+N金币）】
    5. 新装备词条高于旧装备 → 绿色▲；低于 → 红色▼
    6. 已有装备没有的词条 → 仅在新栏显示，无对比箭头

3. **词条系统（24种）**
    
    **核心属性（7种）**：
    - 攻击力（attack）——整数，影响基础伤害
    - 生命值（hp）——整数，3倍基数
    - 防御力（defense）——整数，伤害减免公式：def/(def+200)
    - 穿甲（armorPen）——整数，抵消敌方防御
    - 暴击率（critRate）——百分比，触发暴击的概率
    - 暴击抵抗（critResist）——百分比，抵消敌方暴击率
    - 暴击伤害（critDamage）——百分比，暴击时的额外伤害倍率

    **进阶属性（5种）**：
    - 命中（hit）——百分比
    - 闪避（dodge）——百分比，完全规避伤害的概率
    - 伤害加成（dmgBonus）——百分比
    - 伤害减免（dmgReduce）——百分比
    - 攻击速度（atkSpeed）——小数，额外攻速加成

    **元素属性（8种）**：
    - 火精通/水精通/土精通/风精通（fireMastery等）——百分比，额外元素伤害
    - 火抗/水抗/土抗/风抗（fireResist等）——百分比，减免元素伤害

    **特殊属性（4种）**：
    - 治疗效果（healEffect）——百分比
    - 反击概率（counterRate）——百分比，被攻击时触发50%伤害反击
    - 吸血系数（lifeSteal）——百分比，造成伤害时回复等比例生命
    - 韧性（tenacity）——整数，提升眩晕敌人的概率

4. **词条生成规则**
    1. 每件装备**必定包含**该槽位的主词条（且数值享受2.8倍加成）
    2. 然后从剩余23种词条池中**不放回随机抽取**N-1种，N为品质对应的词条数量
    3. 主词条数值 = 基础值 × 品质倍率 × 2.8
    4. 副词条数值 = 基础值 × 品质倍率 × 1.0
    5. 百分比类词条基础值：2~7；固定值类词条基础值：15~50
    6. 最终数值 = floor(基础值 × (0.85+random×0.3))

5. **战力计算公式**
    - 单件装备战力 = Σ(各词条战力贡献)
    - 百分比词条贡献 = 数值 × 8
    - 攻速词条贡献 = 数值 × 40
    - HP词条贡献 = 数值 × 0.2
    - 其他固定值词条贡献 = 数值 × 1.0
    - 总战力 = Σ(各槽位装备战力)

## 3. 装备升级系统

1. 消耗金币提升**全局装备等级**
2. 升级费用公式：**100 × 1.5^(当前等级-1)**
3. 每升1级，基础属性增加：攻击力+10、生命值+100、防御力+5
4. 基础属性独立于装备词条，是角色脱装后的底线属性
5. 按钮文案显示：「升级装备 Lv.N→N+1 (XXX G)」
6. 金币不足时按钮置灰不可点击

## 4. 关卡战斗系统

1. **关卡规则**
    1. 线性推进，从第1关开始
    2. 每关敌人属性固定，不受玩家状态影响
    3. 通过后关卡+1，失败后关卡不变

2. **敌人属性公式**（第N关）
    - HP = floor(100 × 1.18^(N-1))
    - ATK = floor(15 × 1.14^(N-1))
    - DEF = floor(5 × 1.10^(N-1))
    - 暴击率 = 5% + N × 0.2%
    - 暴击伤害 = 150% + N × 1%
    - 攻速 = 1.0 + N × 0.01
    - 闪避 = min(30%, N × 0.5%)

3. **推荐战力** = floor(50 × 1.16^(N-1))

4. **战斗规则**
    1. 100%自动即时战斗，双方同时对攻
    2. Tick频率：requestAnimationFrame（约60fps），累计伤害按攻速计算攻击间隔
    3. 玩家攻击间隔 = 1000 / (1 + 攻速) ms，最低60ms
    4. 敌人攻击间隔 = 1000 / 敌人攻速 ms，最低80ms
    5. 伤害计算公式见下文

5. **玩家伤害计算**
    - 基础伤害 = 攻击力 × (0.92~1.08随机)
    - 暴击判定：随机数 < 暴击率% → 伤害 × (1 + 暴击伤害%)
    - 伤害加成：伤害 × (1 + 伤害加成%)
    - 穿甲生效：敌方有效防御 = max(0, 敌方防御 - 穿甲值)
    - 防御减免：伤害 × (1 - 有效防御/(有效防御+200))
    - 元素伤害：伤害 + floor((火+水+土+风精通) × 0.3 × 随机)
    - 最终伤害最低为1

6. **敌人伤害计算**
    - 基础伤害 = 敌方ATK × (0.92~1.08随机)
    - 敌方暴击判定：随机数 < (敌方暴击率 - 暴击抵抗%)，暴击则 × 敌方暴伤
    - 闪避判定：随机数 < 闪避% → 本次伤害为0
    - 伤害减免：伤害 × (1 - 伤害减免%)
    - 防御减免：伤害 × (1 - 防御/(防御+200))
    - 元素抗性减免：伤害 × (1 - (火+水+土+风抗性合计)/500)
    - 最终伤害最低为1

7. **特殊机制**
    - **吸血**：每次攻击恢复 floor(造成伤害 × 吸血系数%)
    - **反击**：非暴击时，概率触发50%伤害的反击
    - **眩晕**：玩家攻击时，8%基础概率 × (1+韧性/200)眩晕敌人500ms
    - 眩晕期间敌人跳过攻击计时

8. **战斗结束条件**
    - 我方HP ≤ 0 → **失败**
    - 敌方HP ≤ 0 → **胜利**
    - 超过60秒未结束 → 按HP百分比判胜负（高者胜）

9. **战后统计面板**（6项）
    | 统计项 | 计算方式 |
    |--------|----------|
    | 总伤害 | 玩家每次攻击造成伤害的累加值 |
    | 承受伤害 | 玩家每次受到伤害的累加值 |
    | 治疗量 | 吸血回复的生命值累加 |
    | 暴击次数 | 玩家触发暴击的总次数 |
    | 控制时长 | 眩晕敌人累计的毫秒数 |
    | 战斗时长 | 从开始到结束的实际秒数（精确到0.1s） |
