# 【模板】功能名称

> **模板用途**：配合 HTML 原型验证工作流使用，原型验证通过后、进入正式开发前，由策划填写。本模板是**正式开发策划案**，描述的是一份可直接交付程序/美术/音效/QA 执行的落地文档。
> **来源原型**：`【原型文件路径】`
> **对应 Spec**：`【spec-templates/ 对应文件路径】`
> **对应设计文档**：`【game/product 模板的设计文档路径】`
> **文档状态**：[ ] 草稿 / [ ] 评审中 / [ ] 已确认
>
> **与 game/product 模板的分工**：
> - `game` / `product` 模板 = 设计意图文档，记录 WHY（为什么这样设计 / 验证结论 / 体验目标 / 平衡目标）
> - 本模板 = 落地策划案，记录 WHAT+HOW（系统功能 / 页面 / 埋点 / 动效 / 音效 / 多语言），是进入正式开发的执行依据
> - **正常流程下，一个玩法/系统需要同时产出 game/product 文档 + 本模板共两份**：前者记录设计思考，后者交付执行

#### 文档规范

- 一般内容使用正常字体

- **关键内容使用粗体字**

- **重要的内容使用红色粗体字**

- 最后版本的调整内容使用黄色底色

- 有争议或待确认内容使用红色底色
