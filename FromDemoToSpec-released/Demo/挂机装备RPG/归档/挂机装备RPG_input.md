## 二、必填区

### A. 核心验证目标

**假设**：挂机增量装备游戏中，玩家在"宝箱自动产出→自动开箱→装备择优替换→金币升级→闯关验证"这个闭环中，是否能持续感受到成长的爽快感，且不需要复杂决策就能保持投入？

**验证判据**：
- 试玩者能在 3 分钟内自然理解装备替换逻辑（战力数字一看就懂）
- 试玩者至少主动点击 3 次"手动开箱"加速消耗
- 试玩者能说出"我换了个更好的装备，战力涨了"的时刻
- 战斗结束后试玩者会主动看一眼战后统计

**最担心的翻车点**：数值设计崩了——要么装备掉太快没感觉（边际效用递减），要么反馈不够爽快（数字不够大、不够频繁、不够震）。

### B. 游戏类型与机制描述

**类型**：挂机增量 · 装备驱动 · 关卡验证

**核心循环**：
```
宝箱自动产出（3s/个） → 自动开箱（5s/个，带动画）
        │
        ▼
宝箱堆积（上限100） ← 手动点击加速开箱
        │
        ▼
装备掉落（13槽位任一位，与当前等级匹配）
        │
        ├─ 战力低于当前同部位 → 自动卖出换金币
        │
        └─ 战力高于当前同部位 → 暂停开箱，玩家选择替换/卖出
                │
                ▼
        金币积累 → 升级装备等级 → 所有装备槽统一升级
                │
                ▼
        战力提升 → 挑战关卡验证 → 自动战斗 → 战后统计
```

**核心张力**：挂机产出与手动加速的节奏感——宝箱自动开得慢，堆多了需要你动手；装备来了需要你判断（但判断成本极低——看战力数字就行）；打完关卡看到数字在涨，知道自己变强了。

### C. 核心数值（可配置参数）

| 参数 | 值 | 说明 |
|------|-----|------|
| 宝箱产出间隔 | 3s | 每 3 秒自动生成 1 个宝箱 |

| 宝箱上限 | 100 | 到达上限后停止产出，开启后补产 |
| 装备槽位 | 13 | 帽子/面具/耳环/项链/衣服/裤子/护肩/护膝/手套/鞋子/披风/腰带/武器 |
| 品质档位 | 10 | 普通/常见/稀有/史诗/传说/不朽/闪曜/完美/传世/至尊 |
| 装备等级 | 统一 | 所有槽位共享同一等级，金币统一升级 |

**战力计算**：
```
战力 = Σ(攻击 × 2 + 生命 × 0.5 + 防御 × 1.5 + 暴击率% × 10 + 暴击伤害% × 3 + ...)
（各词条加权求和，品质系数作为总乘数）
```

### E. 经济系统

**资源类型**：
| 资源 | 说明 | 产出来源 | 消耗去向 |
|------|------|----------|----------|
| 宝箱 | 核心产出载体 | 自动生成（3s/个） | 自动开启/手动开启 |
| 装备 | 角色战力载体 | 开宝箱随机掉落 | 装备或卖出 |
| 金币 | 升级货币 | 卖出装备 | 升级装备等级 |

**卖出价格**：`品质系数 × 等级系数 × 基础价格`，基础价格 50 金币。

**升级成本**：`cost(level) = 100 × 1.5^(level-1)`，所有槽位统一升级。

### F. 战斗系统

**关卡设计**：

| 关卡 | 敌人HP | 敌人ATK | 敌人DEF | 推荐战力 |
|------|--------|---------|---------|----------|
| 第1关 | 500 | 20 | 5 | 200 |
| 第2关 | 1,200 | 40 | 10 | 500 |
| 第3关 | 2,500 | 75 | 20 | 1,200 |
| 第4关 | 5,000 | 140 | 40 | 2,800 |
| 第5关 | 10,000 | 260 | 80 | 6,500 |

**战斗逻辑**：
- 自动回合制，每 300ms 一 tick
- 双方同时攻击：玩家伤害 = `max(ATK × (1+伤害加成%) - 敌DEF × (1-穿甲%), 1)` × 暴击判定 × 攻速修正
- 敌方伤害 = `max(敌ATK - 玩家DEF, 1)` × 命中/闪避判定
- 吸血：造成伤害 × 吸血系数 → 回复玩家 HP
- 反击：受击时概率触发，造成 50% 普通伤害
- 韧性：减少敌方控制效果

**战后统计**：
- 总伤害、受到伤害、暴击次数、治疗量、通关时长

**战斗表现**：
- 数字实时弹出（普通白→暴击黄大字→治疗绿字）
- 击杀时震屏
- 通关时大型反馈

### G. 反馈与爽快感设计

**震屏触发时机**：
- 获得战力更高的装备替换时（轻震）
- 战斗中暴击时（微震）
- 战斗中击杀敌人时（中震）
- 关卡通关时（大震）
- 获得高品质装备（史诗及以上）时（中震+闪光）

**数字特效**：
- 伤害数字：白色，连续弹出，暴击放大变黄
- 战力数字暴涨：装备替换后战力变化，绿色↑箭头+数字，红色↓箭头+数字
- 金币数字：金色弹出动画
- 宝箱开启：宝箱图标晃动→开启→装备图标飞入槽位

**节奏控制**：
- 自动开箱间隔 5s（含动画）让每个开箱都有"期待感"
- 手动点击加速是"我想快点看到下一个装备"的主动行为
- 装备替换暂停机制制造"选装备的仪式感"

#### 产出源定义

| 产出源 | 产出内容 | 基础产出量 | 产出间隔 |
|--------|----------|-----------|----------|
| 自动宝箱生成器 | 宝箱×1 | 1 | 3s |
| 自动开箱器 | 装备×1（随机槽位+品质+词条） | 1 | 5s |
| 手动开箱 | 同上 | 1 | 即时 |

#### 升级成本曲线

```
cost(level) = 100 × 1.5^(level-1)
```

| 等级 | 成本 | 累计 |
|------|------|------|
| 1→2 | 100 | 100 |
| 2→3 | 150 | 250 |
| 3→4 | 225 | 475 |
| 4→5 | 338 | 813 |
| 5→6 | 506 | 1,319 |
| 10→11 | 5,767 | 11,333 |

#### 通胀安全阀

- 宝箱上限 100（防止无限堆积）
- 装备自动比较战力（防止无效装备堆积）
- 升级成本指数增长（防止后期失速换为线性停滞）
- 验证窗口：5 个关卡，约在 5-10 分钟游戏时长内完成验证

### [✓] 战斗/对抗类

#### 对战模式

PvE（玩家装备 vs 固定数值关卡敌人），全自动观战。

#### 战场规格

- 类型：数值对撞（无空间位置）
- 表现形式：双方血条实时变化 + 伤害数字弹出

#### 战斗 tick 结构

```
每 300ms：
1. 玩家攻击：计算伤害 → 判定暴击 → 判定吸血 → 弹出伤害数字
2. 敌方攻击：计算伤害 → 判定闪避 → 判定反击 → 弹出伤害数字
3. 更新双方 HP 条
4. 检查胜负：任一方 HP ≤ 0 → 战斗结束 → 弹出战后统计
```

#### 伤害公式

```
玩家伤害 = max(ATK × (1 + 伤害加成%) - 敌DEF × (1 - 穿甲%), 1)
暴击判定：随机 < 暴击率 → 伤害 × 暴击伤害倍率
吸血回复 = 玩家伤害 × 吸血系数
敌方伤害 = max(敌ATK - 玩家DEF, 1)
闪避判定：随机 < 闪避率 → 敌方伤害 = 0，显示"MISS"
反击判定：闪避成功 且 随机 < 反击概率 → 造成 敌伤害 × 0.5
```

#### 胜负条件

- 胜利：敌人 HP ≤ 0 → 解锁下一关
- 失败：玩家 HP ≤ 0 → 提示"战力不足，继续挂机提升装备"
- 统计展示：总伤害、受到伤害、暴击次数、治疗量、通关时长

## 六、附录：AI 备注

1. **爽快感是核心**：设计者反复强调"反馈一定要做好""爽快感做出来"，因此 HTML 原型必须在数字特效、震屏、色彩反馈上下重功夫。这是最可能翻车但也最重要的点。
2. **简化决策**：设计者明确表示"玩家基本不用做选择"，唯一决策点是装备替换——而这个决策已经被简化为"看战力数字"。所有其他设计都要服务于"降低认知负荷"。
3. **节奏控制关键**：自动开箱 5s/个是有意拉长——让玩家"想更快"然后动手点，而不是让系统无脑快。这个节奏感是该游戏区别于纯挂机的核心差异。
4. **设计者提到"开到比现有的战力更低的就直接卖掉"**，这意味着装备系统不需要背包——只有"当前穿戴的"和"卖掉的"。这个设计哲学（所见即所得）是很好的简化方向。
