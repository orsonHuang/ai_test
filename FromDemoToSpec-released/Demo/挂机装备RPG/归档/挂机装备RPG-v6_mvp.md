| 开箱速率公式 | 2500/(1+ct×0.08) | **max(1500, 2500-ct×125)**：每升一档减 125ms，下限 1500（追平产出，不反超） |

| 自动替换解锁 | - | **AUTO_EQUIP_UNLOCK = 1000**（c3c 从 500 调升）。前期手动抉择更多，避免免确认提前介入冲淡装备体验 |
| 区域 UI | 无 | **顶部 panel 显示「🗺️ 区域 X/10」**（区域 = floor(cSt/10)+1）。每 10 关一区域，BOSS 首通后飘字 "🎉 解锁第 N 区域！"（c3b 新增） |
| calcPower 战力 | hp 词条权重 0.5 | **hp 权重 0.3**（c3b 调）。降低 hp 堆叠虚高，让战力数字与实战表现更"诚实" |

**A2 节奏机制详解**：
- ct=0: 开箱 2.5s/产出 1.5s → 净累积 1 箱/2.5s（约 24 箱/分钟）→ 玩家手动连开爽感来源
- ct=4: 开箱 2.0s/产出 1.5s → 仍累积，但速度变慢
- ct=8: 开箱 1.5s/产出 1.5s → **追平**，库存停止增长
- ct=9: max(1500, 2500-1125)=1500 → 不反超，库存价值保留
- 设计意图：前期"积累→爆开"，中期"持续产出"，后期"挂机价值=进战斗+宝箱档位升级"，避免后期"几百个箱子要开"的负担感（试玩反馈 #10）

## 一、功能概述

### 1.1 原型名称

挂机装备RPG（Idle Equipment RPG）—— 挂机增量装备驱动，含宝箱品质成长 + 装备独立升级 + 宝石经济 + 关卡数值验证

### 1.2 核心体验

玩家进入游戏后，宝箱自动以每 1.5 秒 1 个的速度产出，并以每 2.5 秒 1 个的速度自动开启。手动开箱功能初始锁定（需开启20箱解锁），装备槽位分列解锁（开40/100/200/500/1000箱逐步解锁更多列）。宝箱开启时掉落随机已解锁槽位的装备（共 14 个槽位，7列×2行），品质受宝箱品质限制。战力达标后点击"进入战斗"挑战关卡，首通奖励宝箱（可超上限），复刷奖励金币。

核心特色：**极简决策 + 装备独立成长 + 宝箱品质纵深 + 极致反馈**。

### 1.3 验证目标

| 项目 | 内容 |
|------|------|
| **核心假设** | 挂机增量装备游戏中，"自动产出→偶尔手动加速→战力数字看板→简单替换决策→独立升级→宝箱品质成长→战斗验证"的闭环，能否让玩家在极低认知负荷下持续感受到成长爽快感 |
| **验证判据** | 1) 试玩者 3 分钟内理解装备替换和独立升级；2) 试玩者至少主动点击 3 次手动开箱；3) 试玩者能说出"我换了更好的装备"的时刻；4) 战斗后看战后统计；5) 宝石掉落时有惊喜感 |
| **不验证** | 美术风格、音效、联网功能、长期养成系统（百级以上）、元素精通/抗性系统（三档词条）、PvP、社交分享、移动端适配 |

## 二、核心设计

### 2.1 场景/界面规格（v2 重排）

**布局（从上到下）**：
1. **玩家属性面板**（顶部，蓝色边框）：战力、攻击、生命、防御、暴击率、暴伤、穿甲、命中、闪避、伤害加成、伤害减免、吸血、攻速、反击 —— 14 项实时显示
2. **装备槽位网格**（7列×2行 = 14槽，v5新排列）：
   - 第1排：武器/帽子/手套/护肩/副手武器/面具/耳环
   - 第2排：衣服/裤子/鞋子/护膝/披风/腰带/项链
   - 锁定槽位显示🔒+解锁条件（仅下一列），未解锁列不显示条件
   - 已解锁空槽位显示"空"，有装备槽位显示Lv标签/emoji/战力/基础属性/词条/升级按钮
3. **动态区域**（宝箱/抉择/战斗三区互斥）：
   - 默认为宝箱区：数量+品质标签+进度条+升级宝箱/开箱/进入战斗按钮
   - 抉择时替换为对比对话框（新旧装备战力+词条+Lv对比）
   - 战斗时替换为战斗区（关卡选择+HP条+数字区+挑战+返回按钮）

**左侧固定**：200px 调试面板（仅热调滑块+一键重置+日志）

### 2.2 时序/流程结构

```
游戏启动
  │
  ├─→ 宝箱自动产出（3s/个，上限 100）—— 不受阶段影响持续运行
  │     │
  │     ├─→ 自动开箱（5s/个，仅 idle 阶段执行）
  │     │     │
  │     │     ├─→ 宝石掉落判定（5%）→ 💎 震屏闪光
  │     │     │
  │     │     └─→ 装备掉落（品质受宝箱品质限制）
  │     │           │
  │     │     ┌─────┴─────┐
  │     │   更低/相等    更高
  │     │     │           │
  │     │  自动卖出      暂停开箱 → 装备抉择
  │     │  +金币              │
  │     │                 ┌───┴───┐
  │     │              替换装备  放弃(卖出)
  │     │                 │
  │     │                 ▼
  │     │            继续开箱
  │     │
  │     └─→ 宝箱堆积 → 手动点击加速
  │
  ├─→ 金币 → 独立升级装备（每件独立按钮，50×1.5^(Lv-1)）
  │
  ├─→ 宝石 → 升级宝箱品质（10档，解锁更高品质装备区间）
  │
  └─→ 战力达标 → 点击"进入战斗"
        │
        ▼
    战斗区域显示（宝箱区隐藏，开箱冻结，产出继续）
        │
        ▼
    全自动战斗（300ms/tick）→ 通关/失败
        │
        ▼
    战后统计 → 点击"返回挂机" → 宝箱区恢复
```

### 2.3 装备系统详细设计

#### 装备槽排列（v5，7列×2行）

```
Row1: ⚔️武器(0)  🎩帽子(1)  🧤手套(2)  🛡️护肩(3)  🗡️副手武器(4)  🎭面具(5)  💎耳环(6)
Row2: 👘衣服(7)  👖裤子(8)  👟鞋子(9)  🦿护膝(10) 🧣披风(11)  🔗腰带(12) 📿项链(13)
```

#### 装备槽解锁（v5）

| 列 | 槽位 | 解锁条件 | 说明 |
|----|------|----------|------|
| 1列(0,7) | 武器/衣服 | 初始解锁 | - |
| 2列(1,8) | 帽子/裤子 | 初始解锁 | - |
| 3列(2,9) | 手套/鞋子 | 总开箱40 | 显示解锁倒计时 |
| 4列(3,10) | 护肩/护膝 | 总开箱100 | 仅3列解锁后显示 |
| 5列(4,11) | 副手武器/披风 | 总开箱200 | 仅4列解锁后显示 |
| 6列(5,12) | 面具/腰带 | 总开箱500 | 仅5列解锁后显示 |
| 7列(6,13) | 耳环/项链 | 总开箱1000 | 仅6列解锁后显示 |

> 每次开箱 `totalOpened++`，仅下一列显示 `开N箱解锁(X/N)`，其余锁定不显示条件

#### 基础属性（按槽位分类）

| 类别 | 槽位 | 基础属性 | Lv1 值 | 每级增长 | Lv1战力贡献 |
|------|------|----------|--------|----------|------------|
| 攻击型 | 武器、副手武器、手套、面具、耳环、项链 | 攻击 | 5 | +3/级 | 10 |
| 防御型 | 帽子、鞋子、披风、腰带 | 防御 | 7 | +3/级 | 10.5 |
| 生命型 | 衣服、裤子、护肩、护膝 | 生命 | 20 | +10/级 | 10 |

> 设计意图：三类基础属性贡献的战力在设计上持平（LV1均约6点），确保公平性。

#### 装备独立升级

- 每件装备槽位上有独立升级按钮（⬆ + 金币消耗）
- 升级成本：`50 × 1.5^(level-1)` 金币
- 升级效果：基础属性按类别增长，战力重新计算
- 金币不足时按钮置灰
- 升级仅改变基础属性和战力，不改变品质和词条

#### 战力公式（保持加权和，c4 重写已回退）

```
单件 calcPower(eq):
  power = baseStatPow(槽位类型, lv, q)   # 槽位基础属性 × 类型权重
        + Σ (词条值 × 词条权重)
  
  其中：
  - baseStatPow：atk 槽 ×2 / def 槽 ×1.5 / hp 槽 ×0.5
  - 词条权重表见 §3.5（atk×2 / hp×0.3 / def×1.5 / pct 类各异）

总战力 totalPow():
  Σ S.eq[i].p   # 14 件装备单件战力加和
```

**已知数值难题（不在原型阶段解决）**：
- 战力数字与实战表现不完全一致：试玩多次出现"高战力打不过低推荐战力关卡"或"低战力打过高推荐战力关卡"。
- c3b 调 hp 词条权重 0.5→0.3（治标）；c4 系列尝试过几何平均、基于 pStats() 的实战属性公式、量级放大系数等多种重写方案，均出现新问题（数据流脱节 / UI 矛盾 / 量级错位）。
- **决策**：原型阶段不再精调，沉淀此问题到 design-doc / spec 已知约束段。正式立项后由专职数值策划做平衡（需要的工具：批量战斗模拟器跑参数空间，找战力↔通关率拟合曲线）。
- **MVP 校准临时方案**：玩家可通过左侧热调面板的"敌人属性倍率"和"玩家伤害倍率"调整战斗难度，绕过战力公式问题。

### 2.4 宝石经济

- **产出**：开箱有 X% 概率掉落宝石，数量 = [1×2^tier, 3×2^tier] × GEM_QTY_MULT，掉落时全屏震屏+💎✨闪光
- **用途**：升级宝箱品质

### 2.7 核心规则

```
规则 1 · 宝箱产出（v6 锁死）：每 1.5s 自动 +1，上限 200+ct×50，不受阶段影响持续运行；产出速率不再随 ct 变化（v5 为 1500/(1+ct×0.08)，v6 锁死 1500）
规则 2 · 开箱限制：仅 idle 阶段自动/手动开箱；combat/choice 阶段进度冻结
规则 3 · 装备掉落：随机槽位，品质受宝箱品质区间限制，概率指数衰减
规则 4 · 战力比较：≤ 当前 → 自动卖出；> 当前 → 暂停开箱弹出抉择
规则 5 · 独立升级：每件装备独立等级，基础属性随等级增长，成本 50×1.5^(Lv-1)
规则 6 · 宝石掉落：开箱 X% 概率掉落宝石，数量 = [1×2^tier, 3×2^tier] × GEM_QTY_MULT，随宝箱品质指数增长
规则 7 · 宝箱升级：宝石消耗，每档限制可开出品质区间
规则 8 · 战斗时序：300ms/tick，攻速修正，双方同时输出
规则 9 · 伤害公式：玩家伤害 = max(ATK×DMG_MULT×(1+dmgBonus%) - 敌DEF×(1-pen%), 1)；敌方伤害 = max(敌ATK - 玩家DEF, 1)×(1-dmgReduce%)
规则 10 · 暴击/闪避/反击：暴击 = 伤害×暴伤倍率；闪避 = 敌方MISS；闪避+反击 = 50%伤害
规则 11 · 命中机制（v4）：上限200%，有效命中 = 命中 - 敌闪避(敌最高30%)，命中率clamp 5-95%
规则 12 · 难度波（v4→v6-c3）：5关精英(+50%)、10关BOSS(+100%)、其余普通关(×1.0)。v6-c3 删除喘息关概念（v5: -35%，c1: -25%，c3: 直接删除恢复普通关）
规则 13 · 关卡奖励（v4→v5→v6）：v5 改首通宝箱（可超上限）、复刷取消；v6 去掉首通金币只剩宝箱
规则 14 · 战斗中返回（v4）：点击返回直接中断战斗
规则 15 · 手动开箱锁定（v5）：初始不可用，总开箱≥20后解锁
规则 16 · 装备槽列解锁（v5）：分7列，总开箱达40/100/200/500/1000逐步解锁
规则 17 · 宝箱溢出（v5）：通关等方式增加的宝箱可超过上限，仅定时产出受限
```
