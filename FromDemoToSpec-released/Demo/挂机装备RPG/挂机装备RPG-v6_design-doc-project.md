# 简介

### 设计目的

1. 打造一个极低认知负荷的挂机装备驱动 RPG，核心体验为：挂机→开箱→换装→升级→挑战关卡的无限循环
2. 验证"挂机增量装备驱动"能否在极低认知负荷下持续提供成长爽感
3. 通过渐进解锁(手动开箱→列解锁→自动替换)维持长期驱动力

### 设计内容

1. 宝箱系统：定时产出+自动开启+手动加速
2. 装备系统：14槽位7列×2行、10档品质、15种词条、独立等级升级
3. 品质成长：10档宝箱品质升级(消耗宝石)、限制可掉落品质区间
4. 关卡战斗：100关 sawtooth波(精英/BOSS/喘息)、全自动即时制战斗
5. 解锁系统：手动开箱(20箱)、列解锁(40/100/200/500/1000)、自动替换(500箱)
6. 经济系统：金币(卖装备/升级)、宝石(升级宝箱)、宝箱(开箱/关卡奖励)

### 名词解释

| **名词** | **说明** |
|----------|----------|
| 战力 | 装备综合数值，由基础属性+词条值按权重合成 |
| 品质 | 10档装备品质，影响战力区间和词条上限 |

# 方案概述

挂机装备RPG 是一款单页面 HTML 原型，布局为：左侧200px调试面板 + 右侧主区域(玩家属性→装备网格→动态区域)。

**核心流程**：
1. 进入游戏，20个宝箱已就位；自动每1.5s产出1个宝箱，每2.5s自动开启1个
2. 开箱获得装备(随机已解锁槽位)，与当前装备比较战力
3. 战力更高则弹出抉择，战力更低自动卖出+金币
4. 装备可独立升级(花金币)，总战力提升后挑战关卡
5. 关卡首通奖励宝箱(可超上限)，宝箱品质升级（花宝石）解锁更高品质装备
6. 随着总开箱数增长，逐步解锁手动开箱、更多装备列、自动替换功能

# 设计决策理由

| 决策点 | 选择了什么 | 排除了什么 | 理由 | 对实现的约束 |
|--------|-----------|-----------|------|-------------|
| 战斗模式 | 全自动即时制 | 手动操作战斗 | 降低认知负荷，核心体验在装备驱动而非操作 | 需要自动战斗引擎+计时器 |
| 关卡不可复刷 | 每关仅首通一次 | 可重复挑战 | 避免"刷关"疲劳，推动玩家持续挂机成长 | 首通后关卡按钮置灰 |
| 装备独立升级 | 每件装备独立等级 | 全局统一等级 | 增加装备差异化和养成深度 | 每件装备需存储独立等级 |
| 分列渐进解锁 | 7列分5个里程碑 | 全部初始解锁 | 持续制造"马上能解锁"的期待感 | 需追踪总开箱数+解锁状态 |
| 品质影响基础属性 | 品质系数1+q×0.25 | 品质只影响词条 | 让品质差距更直观 | baseStat()需传入品质参数 |

# 系统功能

## 宝箱系统

1. **定时产出**
   - 每 CHEST_GEN_BASE/(1+tier×0.08) ms 自动 +1 宝箱
   - 达到上限(200+tier×50)时定时产出暂停
   - 产出不受阶段影响，战斗中/抉择中仍继续

2. **自动开箱**
   - 仅 idle 阶段执行，每 CHEST_OPEN_BASE/(1+tier×0.08) ms 开1个
   - combat/choice 阶段进度条冻结暂停
   - 开箱消耗1个宝箱，totalOpened++，随机已解锁槽位生成装备
   - 5%概率触发宝石掉落，数量 = [1,3]×2^tier×GEM_QTY_MULT

3. **手动加速开箱**
   - 初始锁定，需 totalOpened≥20 解锁
   - 点击按钮消耗1个宝箱立即开箱
   - 与自动开箱共享冷却

4. **关卡宝箱奖励**
   - 首通关卡 addChests(N)，可超过储存上限
   - 奖励 = 20+floor(关/2)，精英×1.5，BOSS×2

## 装备系统

1. **槽位排列（v5新布局）**
   ```
   Row1: 武器(0) 帽子(1) 手套(2) 护肩(3) 副手武器(4) 面具(5) 耳环(6)
   Row2: 衣服(7) 裤子(8) 鞋子(9) 护膝(10) 披风(11) 腰带(12) 项链(13)
   ```

2. **基础属性（按槽位分类，含品质缩放）**
   | 类别 | 槽位 | 公式 |
   |------|------|------|
   | 攻击 | 武器/副手武器/手套/面具/耳环/项链 | (5+(Lv-1)×3)×(1+q×0.25) |
   | 防御 | 帽子/鞋子/披风/腰带 | (7+(Lv-1)×3)×(1+q×0.25) |
   | 生命 | 衣服/裤子/护肩/护膝 | (20+(Lv-1)×10)×(1+q×0.25) |

3. **品质体系（10档）**
   | 品质 | 颜色 | 词条上限 | Lv1战力区间 |
   |------|------|----------|------------|
   | 普通 | #9e9e9e | 1 | 10-28 |
   | 常见 | #4caf50 | 2 | 28-65 |
   | 稀有 | #2196f3 | 2 | 65-140 |
   | 史诗 | #9c27b0 | 3 | 140-280 |
   | 传说 | #ff9800 | 3 | 280-520 |
   | 不朽 | #f44336 | 4 | 520-950 |
   | 闪曜 | #c0a060 | 4 | 950-1650 |
   | 完美 | #e8e8e8 | 5 | 1650-2700 |
   | 传世 | #ffd700 | 5 | 2700-4200 |
   | 至尊 | 彩虹渐变 | 6 | 4200-6800 |

4. **15种词条（v5去韧性）**
   | ID | 名称 | 类型 | Lv1值域 |
   |----|------|------|---------|
   | atk | 攻击 | flat | 3-40 |
   | hp | 生命 | flat | 15-150 |
   | def | 防御 | flat | 2-20 |
   | pen | 穿甲 | pct | 1-8% |
   | crR | 暴击率 | pct | 1-5% |
   | crD | 暴伤 | pct | 5-25% |
   | crS | 暴抗 | pct | 1-5% |
   | hit | 命中 | pct | 1-5% |
   | dod | 闪避 | pct | 1-2% |
   | db | 伤害加成 | pct | 1-8% |
   | dr | 伤害减免 | pct | 1-6% |
   | ls | 吸血 | pct | 1-4% |
   | spd | 攻速 | pct | 2-15% |
   | hb | 治疗加成 | pct | 3-15% |
   | ctr | 反击 | pct | 1-5% |

5. **战力计算**
   ```
   战力 = baseStatPow(槽位, 等级, 品质) + Σ(词条值 × 权重)
   战力范围 clamp 到品质区间
   ```

6. **装备抉择**
   - 掉落战力 > 当前同槽位战力 → 弹出抉择对话框
   - 战力 ≤ 当前 → 自动卖出(金币 = 50×(1+q×0.8)×(1+Lv×0.15))
   - 抉择选项：替换装备(旧装备卖出+金币) / 放弃(新装备卖出+金币)

7. **装备独立升级**
   - 每件装备有独立升级按钮
   - 成本：50 × 1.5^(Lv-1) 金币
   - 效果：基础属性按类别增长，战力重新计算

## 解锁系统

1. **列解锁**（7列，以 totalOpened 驱动）
   | 列 | 包含槽位 | 需开箱 | 显示 |
   |----|---------|--------|------|
   | 1-2 | 武器/帽子/衣服/裤子 | 0 | 初始解锁 |
   | 3 | 手套/鞋子 | 40 | 显示倒计时 |
   | 4 | 护肩/护膝 | 100 | 仅3列解锁后显示 |
   | 5 | 副手武器/披风 | 200 | 仅4列解锁后显示 |
   | 6 | 面具/腰带 | 500 | 仅5列解锁后显示 |
   | 7 | 耳环/项链 | 1000 | 仅6列解锁后显示 |

2. **手动开箱**：totalOpened ≥ 20 解锁
3. **自动替换**：totalOpened ≥ 500 解锁
4. **锁定槽位**：显示🔒，仅下一列显示解锁条件文本，未解锁列仅🔒无文字
5. **已解锁空槽位**：显示"空"
6. **开箱时仅从已解锁槽位中随机选取投落

## 战斗系统

1. **关卡生成**
   ```
   baseRec = 250 × 1.065^关
   modifier: 10尾BOSS×2.0 | 5尾精英×1.5 | >10关的1尾喘息×0.65
   敌HP = rec×0.55, 敌ATK = rec×0.07, 敌DEF = rec×0.012  // v5.1 战斗系数
   敌闪避 = min(20, 关×0.2)%  // v5.1: 上限 30→20，增长率 0.3→0.2
   玩家初始 HP = 200  // v5.1: 500 → 200
   首通宝箱 = 8+floor(i/3) (精英×1.5, BOSS×2)  // v5.1 调降
   ```

2. **战斗流程**
   - 进入战斗视图 → 选择关卡 → 点击"挑战" → 全自动战斗
   - 每 tick(300/(100+攻速%)ms) 双方同时攻击
   - 伤害公式：`max(ATK×DMG_MULT×(1+dmg%) - 敌DEF×(1-pen%), 1)`
   - 暴击判定 → 暴伤倍率；闪避判定 → MISS；暴击震屏；吸血回溯
   - 任一方 HP≤0 → 结束
   - **赢**：首通给金币+宝箱，按钮变"进入下一关" → 点击跳下一关
   - **输**：按钮变"重试"
   - 战斗中可随时点击"返回挂机"中断

3. **关卡列表**
   - 显示10关窗口(已通关1关 + 当前 + 未来8关)
   - 已通关显示"✅已通关"且不可点击
   - 未通关显示"首通+宝箱数"
   - 精英关加⭐前缀，BOSS关加💀前缀

## 宝箱品质升级

1. 10档宝箱品质，消耗宝石升级
2. 每档限制可开出的装备品质区间([max(0,tier-2), min(tier+2,9)])
3. 升级同时提升：宝箱上限(+50/档)、产出速度(间隔/(1+tier×0.08))、开启速度
4. 宝石升级消耗：[0,10,25,60,150,350,800,1800,4000,9000]

## 自动替换

1. 勾选框开关，位于宝箱界面（抉择界面同步显示）
2. 初始🔒，需 totalOpened ≥ 500
3. 开启后，每次抉择触发时自动倒数(AUTO_EQUIP_DELAY ms，默认3s，热调1-10s)
4. 倒数结束自动执行替换(相当于点击✅替换)
5. 退出抉择/进入战斗时自动停止倒计时

## 已知数值难题（v6 沉淀，留给数值策划专项处理）

### 问题描述

战力数字与实战表现不完全一致——v6 试玩多次出现以下场景：

- 玩家战力 1 万，打过推荐 2.5 万的关卡（高估了关卡难度，玩家以为打不过）
- 玩家战力 4820，打不过推荐 1175 的 20 关 BOSS（高估了玩家实力）
- 单件装备战力 45，全身总战力 29（量级错位，部分小于整体）

### 已尝试的方案与结论

| 版本 | 方案 | 结论 |
|------|------|------|
| v5.1+ | 战斗系数调整（HP 200/敌HP×0.55/敌ATK×0.07/敌DEF×0.012）| 缓解了"免疫挨打"和"突然卡关"，但战力↔实战仍不一致 |
| v6-c3b | hp 词条权重 0.5→0.3 | 治标不治本——加权和公式本质问题在于无法反映"暴击/吸血/攻速"的复利效果 |
| v6-c4 | calcPower / totalPow 改几何平均 `(atk×def×hp/10)^(1/3) × (1+Σ pct)` | 与 pStats() 战斗实际属性两套数据流脱节，战力数字与实战完全不挂钩 |
| v6-c4d | totalPow 改用 pStats 实战属性 `sqrt(atkEff × survEff) × utilMult` | 量级与单件 calcPower 错位（总战力 29 < 单件 45）；STAGE_BASE_REC 同步从 250→30 后又出"2 件装备打通 10 关" |
| v6-c4e | totalPow ×10 + STAGE_BASE_REC 30→100 量级校准 | "战力 4820 打不过推荐 1175 BOSS" 仍存在——多套校准均未根除问题 |
| v6-c4f（最终决策）| **回退到 c3c 的加权和公式** | 承认"战力↔实战一致"是数值精调的硬骨头，原型阶段无法靠手动调参解决 |

### 根本原因（c4 系列试错后归纳）

1. **战力公式只有一组数据流，实战公式有多组复利**：战斗实际由 atk × 暴击期望 × 命中率 × 攻速 × 吸血 × 防御折算等多维复利构成，单一加权和或几何平均都无法精确反映。
2. **关卡推荐战力 (`rec`) 与玩家战力是"两个公式独立估算"**：`rec = 250 × 1.065^i × mod` 是数学模型，玩家战力是装备词条加权和——两者用不同维度，对比意义有限。
3. **战力数字没有"分阶段校准"**：v5 加权和的权重表（atk×2 / hp×0.5 / pen×8 等）是经验拍脑袋值，没有用大规模战斗模拟验证拟合曲线。

### 留给后续数值策划的工作

1. **建批量战斗模拟器**：参数化跑 1000+ 场战斗（不同装备组合 vs 不同关卡），统计通关率，做战力↔通关率回归。
2. **用回归结果反推"权重表"**：让加权和公式逼近"实战预期 DPS × 实战预期生存"的简化估算。
3. **关卡推荐战力 `rec` 公式重新校准**：基于上一步的拟合曲线，让"推荐战力 == 50% 通关率门槛"成立。
4. **保留 MVP 阶段的临时缓解**：左侧热调面板的"敌人属性倍率"和"玩家伤害倍率"滑块作为玩家自校准入口。

> **关键约束**：原型阶段的多次手动精调（c3b/c4/c4b/c4c/c4d/c4e）证明，没有量化数据支持的纯手动调参在战力↔实战一致性上是低效的。立项后必须先建模拟器再调参，不要重蹈原型阶段的覆辙。
