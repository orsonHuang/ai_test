# 通用原型描述模板 · 战斗类填写示例

> **用途**：本文件是 `01-prototype-input-template.md` 的战斗类完整填写参考。
> **对应 Spec 模板**：`spec-templates/spec-template-combat.md`
> 参照游戏类型：战棋对抗（2 人 PvP，交替回合制，6×6 网格）。
> 读者可对照本文件理解"填到什么程度 AI 才能动"。
> **填写难度信号**：本示例填写完整度 = 100%。如果你看完觉得「自己填不到这个程度」，建议改走 `01-input-elicitation-protocol.md` 的问答补齐流程——AI 会通过对话帮你补齐。

## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  玩家是否会在"进攻"和"防守型部署"两种战术路线之间产生有意义的取舍——即游戏不会被某一路线统治，两种选法都有赢面。

- **验证成功的判据是什么？**
  连续试玩 6 局，进攻型和防守型各赢 ≥2 局；或在 3 位不同试玩者中均出现"我尝试换了打法"的行为。

- **不需要验证什么？**
  不需要验证：美术风格、音效、长期成长系统、联网功能、移动端适配。

---

### B. 核心交互

- **用户做什么操作？**
  点击己方单位（高亮显示可走格子）→ 点击目标格子（单位移动）→ 若目标格相邻有敌人，自动进入攻击选择 → 确认攻击

- **系统如何响应？**
  移动：格子高亮消失，单位贴图移入新格 → 攻击：弹出伤害数字，更新双方 HP 条 → HP 归零：单位消失，播放击杀日志 → 检查胜负条件

- **交互循环（用箭头表示）**
  ```
  玩家选择单位 → 选择目标格 → 单位移动/攻击 → 检查胜负 → （未结束）→ 切换到对方回合 → 对方行动 → 切回玩家回合
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Unit | id, name, hp, maxHp, atk, def, moveRange, attackRange, team, hasActed | 属于 Player（team=0）或 AI（team=1）；位于 Grid 某个 Cell |
| Cell | x, y, terrain, occupant | 组成 Grid；occupant 指向 Unit 或 null |
| Player | team, units[], actionCount | 持有若干 Unit；每回合 actionCount 重置为单位数 |
| GameState | currentTurn, phase, turnNumber, winner | 全局唯一；phase = 'select'/'move'/'attack'/'end' |

---

### D. 状态与流程

- **系统有哪些状态？**
  `idle`（等待开始）→ `playerTurn`（玩家选择/行动）→ `aiTurn`（AI 行动，自动执行）→ `settlement`（判定胜负）→ `gameOver`（展示结果）

- **状态之间如何切换？**
  ```
  idle           → （点击"开始"按钮）          → playerTurn
  playerTurn     → （所有己方单位已行动）        → settlement
  settlement     → （无胜负）                   → aiTurn
  settlement     → （有胜负）                   → gameOver
  aiTurn         → （AI 行动完毕）              → settlement
  gameOver       → （点击"重置"按钮）           → idle
  ```

- **初始状态**：`idle`，双方各 3 个单位部署在各自底线。
- **结束状态**：`gameOver`，展示胜方名称 + 剩余单位 HP 总和。

---

### E. 核心规则

```
- 规则1：每回合每个单位只能行动一次（移动+攻击，或仅移动，或仅攻击）
- 规则2：近战单位攻击范围 = 1（四方向相邻格），远程单位攻击范围 = 2（但不能攻击相邻格）
- 规则3：伤害公式：damage = max(1, ATK × CONFIG.DAMAGE_MULT - DEF × 0.5)
- 规则4：单位 HP ≤ 0 时立即从棋盘移除
- 边界条件（必须不能发生的事）：
    - 不允许移动到已有己方单位的格子
    - 不允许攻击己方单位
    - 不允许在 gameOver 状态下继续行动
- 特殊情况处理：
    - 平局：双方同一回合内同时消灭对方最后单位 → 提示"平局"，不计入胜负
    - 死锁：双方都无法攻击对方 → 触发 MAX_TURNS 上限，比较双方剩余 HP 总和，高者获胜
```

---

### F. UI 骨架

- **整体布局**：
  左侧 200px 固定面板（调试/状态）+ 中央 6×6 棋盘（每格 80px）+ 右侧 200px 可调参数区

- **关键视觉元素位置**：
  - 单位：贴在格子内，圆形图标，颜色区分阵营（蓝=玩家，红=AI）
  - HP 条：单位图标正上方，绿→黄→红渐变
  - 行动状态：已行动单位图标半透明（opacity 0.5）
  - 可移动格：淡蓝色高亮；可攻击格：淡红色高亮
  - 当前回合提示：棋盘正上方居中文字

- **信息展示优先级**：
  1. 当前回合（谁的回合）
  2. 被选中单位的属性（HP/ATK/DEF）
  3. 双方剩余单位数
  4. 回合计数器

---

### G. 可调参数（调试面板数据源）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| DAMAGE_MULT | 1.0 | 0.5 ~ 3.0 | 全局伤害倍率 |
| UNIT_HP | 100 | 50 ~ 200 | 所有单位的基础 HP（重置游戏生效） |
| AI_DELAY_MS | 600 | 0 ~ 2000 | AI 每步行动的延迟时间（ms），0=瞬间 |
| MAX_TURNS | 20 | 5 ~ 50 | 最大回合数，超出比 HP 总和 |
| CRIT_CHANCE | 0.1 | 0.0 ~ 0.5 | 暴击概率（暴击=伤害 ×2） |

## 选填区：战斗/对抗类

### 对战模式

PvE（玩家 vs AI），AI 控制红方，玩家控制蓝方。

### 回合结构

每回合的步骤（按顺序）：

```
1. 重置所有己方单位的 hasActed = false
2. 玩家（或 AI）依次操作单位（移动 + 攻击）
3. 所有单位 hasActed = true 后，进入结算
4. 结算：检查胜负条件（checkWin()）
5. 无结果：切换 currentTurn，进入对方回合
```

### 战场规格

- 维度：网格（正方形格，俯视角）
- 尺寸：6 × 6 格
- 特殊地形：无（纯空地，便于验证核心博弈，不引入地形变量）

### 单位/角色模板

| 名称 | 类型 | HP | ATK | DEF | 移速/范围 | 特殊能力 |
|------|------|-----|-----|-----|-----------|----------|
| 战士 | 近战 | 120 | 25 | 15 | 移速2/攻距1 | 无 |
| 弓手 | 远程 | 80 | 30 | 5 | 移速1/攻距2 | 不能攻击相邻格 |
| 法师 | 远程 | 70 | 40 | 3 | 移速1/攻距2 | 范围溅射（攻击时对目标相邻格造成 50% 伤害） |

双方各一套（蓝方/红方各 1 战士 + 1 弓手 + 1 法师）。

### 伤害公式

```
damage = max(1, floor(ATK × DAMAGE_MULT - DEF × 0.5))
暴击（概率 CRIT_CHANCE）：damage × 2
溅射（法师）：相邻格 = floor(damage × 0.5)
```

### AI 行为规则

```
优先级（从高到低）：
1. 若有敌方单位在攻击范围内 → 攻击 HP 最低的敌人
2. 若无攻击机会 → 向最近的敌方单位方向移动（曼哈顿距离最短）
3. 若移动和攻击都做了，标记 hasActed = true，换下一个单位
```

### 胜负条件

```
胜利：消灭对方所有单位（对方 units 数组为空）
失败：己方所有单位被消灭
平局：双方同回合同时清空 → 提示平局
超时：达到 MAX_TURNS → 比较双方剩余 HP 总和，高者胜
```
