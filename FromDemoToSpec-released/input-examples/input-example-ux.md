## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  玩家能否在 10 秒内完成"进入邮件 → 找到未读奖励邮件 → 一键全部领取 → 确认领取成功"的完整流程，且不需要任何引导。

- **验证成功的判据是什么？**
  5 位测试者中，≥4 位能在不看提示的情况下 10 秒内完成全流程；无人误操作"删除"而不是"领取"。

- **不需要验证什么？**
  不需要验证：邮件发送逻辑、服务器存储、推送通知、已读同步、历史邮件归档超过 30 条。

---

### B. 核心交互

- **用户做什么操作？**
  进入邮件页 → 看到邮件列表（未读高亮）→ 点击邮件查看详情 → 点击"领取"按钮 → 或点击"全部领取"批量操作

- **系统如何响应？**
  进入时：自动标记为"已查看"（但奖励未领取）→ 点击"领取"：奖励入背包，邮件状态变"已领取"，按钮灰化 → 点击"全部领取"：批量处理所有有奖励的邮件，顶部提示"已领取 N 件奖励"

- **交互循环**
  ```
  邮件列表 → 点击邮件 → 邮件详情 → 领取/已领取 → 返回列表 → （列表更新状态）
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Mail | id, title, content, type(reward/system/event), reward[], status(unread/read/claimed), expire_time | 组成 MailList |
| Reward | item_id, item_name, quantity, icon | 属于 Mail |
| MailState | mails[], unreadCount, selectedMailId | 全局状态，驱动列表和详情渲染 |

---

### D. 状态与流程

- **系统有哪些状态？**
  `list`（邮件列表页）→ `detail`（邮件详情页）→ `claiming`（领取动画中）→ `empty`（无邮件时的空状态）

- **状态之间如何切换？**
  ```
  list    → （点击某封邮件）           → detail
  detail  → （点击返回）               → list
  detail  → （点击"领取"）             → claiming
  claiming → （领取完成动画结束）       → detail（按钮变灰）
  list    → （所有邮件已读/领取/删除） → empty
  ```

- **初始状态**：`list`，预置 5 封测试邮件（2封未读奖励邮件 + 2封已读系统通知 + 1封活动邮件）。
- **结束状态**：无固定结束（持续可用），极端情况为 `empty`。

---

### E. 核心规则

```
- 规则1：打开邮件详情时，状态自动从"未读"改为"已读"（但奖励不自动领取）
- 规则2：奖励邮件的"领取"按钮仅在 status=read 且未过期时可点
- 规则3：过期邮件（expire_time < 当前时间）：奖励不可领取，标记"已过期"，只可删除
- 规则4："全部领取"仅处理 status=read 且未过期的奖励邮件，系统通知不处理
- 规则5：邮件列表排序：未读在前 → 有奖励在前 → 按时间降序
- 边界条件：
    - 不允许重复领取同一封邮件的奖励（按钮领取后立即灰化）
    - 不允许已过期邮件的奖励被领取
    - 空列表时不显示"全部领取"按钮
- 特殊情况处理：
    - 邮件列表为空：展示空状态图 + "暂无邮件"文字
    - 网络超时（原型中模拟为随机延迟）：显示"领取失败，请重试"toast
```

---

### F. UI 骨架

- **整体布局**：
  左侧邮件列表（固定宽 280px）+ 右侧邮件详情区（自适应宽）+ 顶部"全部领取"按钮

- **关键视觉元素**：
  - 邮件列表项：标题 + 类型图标 + 未读红点 + 日期；已领取邮件半透明
  - 详情区：标题 + 正文 + 奖励展示卡片（图标+名称+数量）+ 领取按钮
  - 全部领取：顶部右侧，仅有未领取奖励邮件时高亮
  - 空状态：居中插图 + "暂无邮件"

- **信息展示优先级**：
  1. 未读数量（顶部徽章，进入前已可见）
  2. 可领取奖励的邮件（列表中优先展示）
  3. 邮件详情正文
  4. 已读/已领取/过期状态标记

## 选填区：产品/UX 原型类

### 用户与场景

- **目标用户角色**：中重度手游玩家，每日登录，习惯处理日常任务
- **使用场景**：每日登录后处理积累的奖励邮件，主要在手机端

### 用户旅程

```
1. 进入 → 主界面看到邮件入口有红点（N条未读）
2. → 点击进入邮件页，看到按未读优先排序的邮件列表
3. → 点击第一封未读奖励邮件，查看内容和奖励详情
4. → 点击"领取"或返回列表后"全部领取"
5. 完成 → 红点消失，退出邮件页
```

### 关键页面/屏幕

| 页面名称 | 展示内容 | 可交互元素 | 进入/退出条件 |
|----------|----------|------------|---------------|
| 邮件列表 | 所有邮件的标题/类型/状态/日期 | 点击邮件、全部领取按钮 | 进入：任意时刻；退出：返回按钮 |
| 邮件详情 | 邮件标题/正文/奖励内容 | 领取按钮、删除按钮、返回按钮 | 进入：点击列表项；退出：返回/侧滑 |
| 空状态 | 空状态插图 + "暂无邮件"文字 | 无 | 进入：列表为空时自动 |

### 交互细节

- 动画/过渡效果：邮件详情从右侧滑入（300ms ease），领取时奖励图标飘向屏幕右上角
- 加载状态展示：列表加载时骨架屏（3行占位）
- 空状态展示：居中插图 + "暂无邮件" + 不显示任何操作按钮
- 错误状态展示：领取失败时底部 toast 提示（2秒自动消失）

### 导航结构

```
[主界面] → （点击邮件入口）→ [邮件列表]
[邮件列表] → （点击某封邮件）→ [邮件详情]
[邮件详情] → （返回按钮/侧滑）→ [邮件列表]
[邮件列表] → （返回按钮）→ [主界面]
```
