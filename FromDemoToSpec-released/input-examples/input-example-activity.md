## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  玩家是否愿意在"稳健得分"和"冒险高分"之间做取舍——即高风险操作（加速、连击）带来的额外分数是否能吸引玩家主动尝试，而不是只用最安全的策略刷分。

- **验证成功的判据是什么？**
  10 局试玩中，至少有 3 局玩家主动使用了"连击"或"加速"操作；或试玩者主动问"怎么才能更高分"。

- **不需要验证什么？**
  不需要验证：美术风格、音效、实际奖励价值（用占位符）、服务器联网、排行榜真实数据。

---

### B. 核心交互

- **用户做什么操作？**
  点击/按键触发操作（拾取目标物）→ 快速连续操作触发连击 → 倒计时结束自动结算

- **系统如何响应？**
  拾取：目标物消失，分数增加，新目标出现 → 连击（3次以内2秒）：播放连击提示，分数×1.5 → 倒计时归零：结算本局分数 → 显示本局分数 + 历史最高分

- **交互循环**
  ```
  倒计时开始 → 玩家点击目标 → 分数增加 → （是否触发连击？）→ 循环至倒计时归零 → 结算
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Target | id, x, y, type(normal/bonus), point_value, spawn_time | 由 SpawnManager 创建，被玩家点击后销毁 |
| SessionState | score, combo, comboTimer, timeLeft, clickCount | 全局单例，每局重置 |
| DailyRecord | date, playCount, bestScore, totalScore | 持久化到 localStorage |

---

### D. 状态与流程

- **系统有哪些状态？**
  `idle`（等待开始）→ `playing`（游戏中）→ `settlement`（结算）→ `gameOver`（展示结果 + 奖励）

- **状态之间如何切换？**
  ```
  idle       → （点击"开始"）           → playing
  playing    → （timeLeft = 0）         → settlement
  settlement → （结算动画结束）          → gameOver
  gameOver   → （点击"再来一局"）        → idle（若未超出每日次数）
  gameOver   → （点击"再来一局"但次数满） → 显示"今日已达上限"不切换
  ```

- **初始状态**：`idle`，分数=0，连击=0，目标随机分布在场地中。
- **结束状态**：`gameOver`，展示本局分数 + 是否刷新最高分 + 奖励档位。

---

### E. 核心规则

```
- 规则1：每局时长 CONFIG.GAME_DURATION 秒（默认60秒）
- 规则2：每次点击 normal 目标 +CONFIG.BASE_SCORE 分，bonus 目标 +CONFIG.BASE_SCORE×3 分
- 规则3：2秒内连续点击 ≥3 次 → 连击激活，得分×CONFIG.COMBO_MULT，连击计数器重置
- 规则4：每日最多参与 CONFIG.DAILY_LIMIT 次，超出后按钮灰化不可点
- 规则5：历史最高分存 localStorage，每局结算时对比更新
- 边界条件（必须不能发生的事）：
    - 不允许 timeLeft < 0
    - 不允许每日次数超出 DAILY_LIMIT
    - 不允许连击计数在一局结束后延续到下一局
- 特殊情况处理：
    - 点击空白处不扣分，不重置连击
    - bonus 目标出现概率 CONFIG.BONUS_RATE（默认20%），不保证每局出现
```

---

### F. UI 骨架

- **整体布局**：
  顶部状态栏（分数 + 倒计时 + 连击状态）+ 中央游戏区（目标物随机分布）+ 底部结算区（每局结束后弹出）

- **关键视觉元素**：
  - 目标物：圆形，normal=白，bonus=金，尺寸 48px
  - 连击提示：点击时浮现 "COMBO×N!" 文字，1秒后消失
  - 倒计时：顶部居中，≤10秒时红色警示
  - 分数：顶部左侧，每次得分时数字跳动动画

- **信息展示优先级**：
  1. 倒计时（剩余时间最关键）
  2. 当前分数
  3. 连击状态（正在连击时高亮显示）
  4. 历史最高分（弱显示在分数旁）

### G. 可调参数（调试面板数据源）

> 最少填 3 个（对应原型编写规范 §2.2 强制要求）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| GAME_DURATION | 60 | 10 ~ 120 | 单局时长（秒） |
| BASE_SCORE | 10 | 1 ~ 50 | normal 目标基础分值 |
| COMBO_MULT | 1.5 | 1.0 ~ 3.0 | 连击得分倍率 |
| BONUS_RATE | 0.2 | 0.0 ~ 0.5 | bonus 目标出现概率 |
| DAILY_LIMIT | 5 | 1 ~ 20 | 每日参与次数上限 |

## 选填区：活动/小玩法类

### 基础信息

- 活动名称：点击狂潮（占位名，验证核心机制用）
- 单局时长：60 秒
- 每日参与次数限制：5 次
- 参与门槛：无

### 得分/积分规则

- 基础得分：每次点击 normal 目标 +10 分
- 加分条件：
  - bonus 目标（金色）：+30 分
  - 连击（2秒内≥3次）：得分 ×1.5
- 扣分条件：无（点空白不扣分）

### 奖励规则

```
按分数区间（占位数值，待验证后调整）：
- ≥ 500 分 → S 级奖励
- 300~499 分 → A 级奖励
- 100~299 分 → B 级奖励
- < 100 分 → 参与奖励（保底）
```

### 社交维度

- 排行榜：无（本地最高分记录，不联网）
- 好友比拼：无
- 分享机制：无

### 活动生命周期

- 活动持续：本次原型不设限，验证核心机制为主
- 阶段变化：无
