## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  初期资源循环是否形成"看得到终点的驱动感"——玩家能在 5 分钟内感知到"再多一点资源就能解锁新东西"，而不是"完全不知道攒什么、攒多少"。

- **验证成功的判据是什么？**
  3 位试玩者中，≥2 位在 5 分钟内主动完成了第一次建筑解锁；且至少 1 位主动调整了资源分配策略（如从"只建一种建筑"改为"多种并建"）。

- **不需要验证什么？**
  不需要验证：长期成长曲线（>30分钟）、PvP 经济博弈、支付/变现设计、30日留存健康度。只验证前5分钟循环驱动感是否成立。

---

### B. 核心交互

- **用户做什么操作？**
  查看当前资源量 → 选择建造/升级建筑 → 等待产出计时器 → 收取资源 → 再次决策建造/升级

- **系统如何响应？**
  确认建造：扣除资源 → 建筑开始计时 → 计时完成 → 自动产出资源 → UI 刷新资源数值；解锁条件满足时 → 高亮新建筑选项

- **交互循环（用箭头表示）**
  ```
  查看资源 → 决策建造 → 扣除成本 → 等待产出 → 资源增加 → 触发解锁检查 → 查看新选项 → 决策建造（循环）
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Resource | id, name, amount, cap | 全局资源池；被 Building.output 写入，被 Building.cost 消耗 |
| Building | id, name, level, outputRate, outputResource, costToBuild, costToUpgrade, unlockCondition | 产出 Resource；受 BuildingSlot 管理 |
| BuildingSlot | id, building(ref/null), isLocked | 持有 Building 引用；格子总数固定 |
| GameState | tick, totalTime, unlockedBuildings[], unlockedResources[] | 全局唯一；驱动 tick 循环 |

---

### D. 状态与流程

- **系统有哪些状态？**
  `idle`（游戏未开始）→ `running`（tick 循环运行中）→ `paused`（调试暂停）→ `milestone`（达成里程碑，短暂提示）→ `ended`（达到验证终点或手动重置）

- **状态之间如何切换？**
  ```
  idle       → （点击"开始"）                   → running
  running    → （点击"暂停"调试按钮）            → paused
  paused     → （点击"继续"）                    → running
  running    → （达成里程碑条件）                → milestone（3s 后自动回 running）
  running    → （点击"重置"）                    → idle
  ```

- **初始状态**：`idle`，初始资源：木材 20、石料 0、粮食 10；初始建筑：伐木场×1（已建）
- **结束状态**：5分钟计时到达，或玩家主动重置，展示资源总收入统计

---

### E. 核心规则

```
- 规则1：每个建筑每隔 TICK_INTERVAL 秒产出一次资源，产出量 = outputRate × level
- 规则2：建筑升级成本随等级指数增长：cost(level) = baseCost × UPGRADE_COST_EXP^(level-1)
- 规则3：新建筑解锁需满足 unlockCondition（资源阈值 or 建筑等级）；满足后自动解锁
- 规则4：资源有上限 cap，超出不产出（倒逼玩家消费资源，避免无限堆积）
- 边界条件（必须不能发生的事）：
    - 资源量不能为负（扣除前必须校验）
    - 同一建筑格不允许重复建造
    - cap 溢出时只丢弃溢出部分，不回滚已有量
- 特殊情况处理：
    - 死锁（所有资源不足以建任何东西）→ 在 UI 高亮提示，允许玩家用"调试注入"功能补充资源
    - 同时满足多个解锁条件 → 批量解锁，统一在下一个 tick 生效
```

---

### F. UI 骨架

- **整体布局**：
  顶部资源栏（3~5 种资源） + 中央建筑格子区（2×3 网格，固定 6 格）+ 右侧建造面板（可建造的建筑列表 + 成本）+ 右下调试面板

- **关键视觉元素位置**：
  - 资源栏：顶部横排，每种资源显示图标+当前量/上限
  - 建筑格：每格显示建筑图标+等级标记；已达产出计时器用进度条显示
  - 建造面板：列出可建造项，已锁定的显示解锁条件；成本用红色标注不可建
  - 里程碑提示：中央弹出横幅，3 秒后淡出

- **信息展示优先级**：
  1. 资源当前量与上限（决策依据）
  2. 各建筑产出进度（倒计时）
  3. 下一个解锁目标的距离（驱动感来源）
  4. 已建建筑等级

---

### G. 可调参数（调试面板数据源）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| TICK_INTERVAL | 5 | 1~30 秒 | 所有建筑的产出间隔（全局） |
| UPGRADE_COST_EXP | 1.8 | 1.2~3.0 | 升级成本的指数增长底数 |
| RESOURCE_CAP_MULT | 1.0 | 0.5~3.0 | 所有资源上限的全局倍率 |
| OUTPUT_RATE_MULT | 1.0 | 0.5~5.0 | 所有建筑产出速率的全局倍率 |
| DEBUG_INJECT_AMOUNT | 50 | 10~500 | 调试"注入资源"按钮每次注入量 |

## 选填区：经济/资源流类

### 资源类型定义

| 资源 ID | 名称 | 初始量 | 上限 | 产出来源 | 消耗去向 |
|---------|------|--------|------|---------|---------|
| wood | 木材 | 20 | 100 | 伐木场 | 建造农场、石矿 |
| stone | 石料 | 0 | 80 | 石矿 | 升级建筑 |
| food | 粮食 | 10 | 60 | 农场 | 解锁人口，驱动后期建筑 |

### 建筑定义

| 建筑 ID | 名称 | 产出 | 基础产出速率 | 建造成本 | 解锁条件 |
|---------|------|------|------------|---------|---------|
| lumber | 伐木场 | 木材 +3/tick | 3 | 初始已建 | 无 |
| farm | 农场 | 粮食 +2/tick | 2 | 木材×15 | 无 |
| quarry | 石矿 | 石料 +2/tick | 2 | 木材×20 | wood.amount ≥ 40（历史峰值） |
| sawmill | 锯木厂 | 木材 +6/tick | 6 | 木材×10 + 石料×10 | quarry 已建 |

### 解锁链路图

```
伐木场（初始）
    │ wood ≥ 40
    ├─→ 石矿 ──────────────→ 锯木厂
    │                           │
    └─→ 农场（wood×15）         └（stone×10 + wood×10）
```

### 通胀安全阀

- **资源上限**：倒逼消耗，不允许无限堆积
- **升级指数成本**：后期升级越来越贵，自然放缓节奏
- **验证窗口（5分钟）**：只验证第一循环，不设计后期失控场景

### 关键数值曲线

```
预期 5 分钟内玩家行为轨迹：
  0s   → 开始，只有伐木场产木材
  30s  → 木材 ≥ 15，可建农场
  60s  → 木材持续累积，农场在产粮食
  120s → 木材峰值达到 40，石矿解锁（关键里程碑）
  180s → 石矿在产，锯木厂条件快满足
  300s → 实验结束，统计各资源总收入
```
