## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  策划能否不依赖程序，独立通过这个工具快速判断"数值调整是否会破坏单位间的平衡"——即工具的输出结果（预期对战回合数/胜率）是否足够直观，策划能直接看懂并做决策。

- **验证成功的判据是什么？**
  3 位策划试用后，能在 5 分钟内独立完成一次"修改某单位 ATK → 观察平衡变化 → 决定是否保留修改"的完整循环，且不需要 AI 解释输出含义。

- **不需要验证什么？**
  不需要验证：实际游戏代码集成、数据库存储、权限管理、多人协同编辑。

---

### B. 核心交互

- **用户做什么操作？**
  在左侧表格中填入/修改单位属性（名称/HP/ATK/DEF） → 点击"模拟对战" → 右侧显示模拟结果

- **系统如何响应？**
  读取表格中所有单位数据 → 用战斗公式两两模拟对战 → 输出每对组合的预期回合数和胜方 → 以颜色区分强弱（红=强，蓝=弱）

- **交互循环**
  ```
  修改单位属性 → 点击"模拟" → 查看结果矩阵 → 发现不平衡 → 调整属性 → 重新模拟
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| Unit | id, name, hp, atk, def | 来自用户输入表格，传入 SimulationEngine |
| BattleResult | unitA_id, unitB_id, winner_id, rounds, hp_remaining | 由 SimulationEngine 输出，传入渲染层 |
| SimulationConfig | CRIT_CHANCE, CRIT_MULT, MIN_DAMAGE | 全局 CONFIG，影响所有模拟计算 |

---

### D. 状态与流程

- **系统有哪些状态？**
  `idle`（等待输入）→ `simulating`（计算中，可能有短暂等待）→ `result`（展示结果矩阵）→ `error`（输入校验失败）

- **状态之间如何切换？**
  ```
  idle       → （点击"模拟对战"，输入合法）   → simulating
  idle       → （点击"模拟对战"，输入非法）   → error
  simulating → （计算完成）                  → result
  result     → （修改任一单元格数据）          → idle（结果矩阵清空）
  error      → （修正输入后）                 → idle
  ```

- **初始状态**：`idle`，预置 3 个示例单位（战士/弓手/法师），结果区空白。
- **结束状态**：无（工具类无结束状态，持续可用）。

---

### E. 核心规则

```
- 规则1：模拟对战使用与游戏原型相同的伤害公式：damage = max(1, floor(ATK - DEF × 0.5))
- 规则2：双方交替攻击，先攻方为当前行的单位（A列 vs B行时，A先攻）
- 规则3：模拟最多进行 CONFIG.MAX_SIM_ROUNDS 回合，超出视为平局
- 规则4：输入校验：HP/ATK/DEF 必须为正整数，名称不能为空
- 规则5：结果矩阵颜色：行单位胜 → 格子为绿色，列单位胜 → 格子为红色，平局 → 灰色
- 边界条件：
    - 单位数量 < 2 不允许模拟（提示"至少需要 2 个单位"）
    - 单位数量上限 CONFIG.MAX_UNITS（默认10），超出提示
    - HP 最大值 CONFIG.MAX_HP（防止无限循环）
- 特殊情况处理：
    - 平局（超出 MAX_SIM_ROUNDS）：格子显示灰色 + "平局" 文字
    - 输入框非数字：边框红色 + 提示"请输入正整数"
```

---

### F. UI 骨架

- **整体布局**：
  左侧单位输入区（表格，可添加/删除行）+ 右侧结果矩阵（N×N 颜色格）+ 顶部 CONFIG 调试区

- **关键视觉元素**：
  - 输入表格：列为 名称/HP/ATK/DEF，行为每个单位，最后一行为"+"添加按钮
  - 结果矩阵：行=进攻方，列=防守方，格子内显示"X回合"，颜色区分胜负
  - "模拟对战"按钮：居中大按钮，计算中显示 loading

- **信息展示优先级**：
  1. 结果矩阵（这是工具的核心输出）
  2. 单位属性表格（输入端，始终可见）
  3. 配置参数（调试面板，可折叠）

---

### G. 可调参数（调试面板数据源）

> 最少填 3 个（对应原型编写规范 §2.2 强制要求）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| MAX_SIM_ROUNDS | 50 | 10 ~ 200 | 单次模拟最大回合数（防止死循环） |
| CRIT_CHANCE | 0.0 | 0.0 ~ 0.3 | 暴击概率（0=关闭暴击系统） |
| CRIT_MULT | 2.0 | 1.5 ~ 3.0 | 暴击伤害倍率 |
| MAX_UNITS | 10 | 2 ~ 20 | 最多允许多少个单位参与模拟 |

## 选填区：系统/工具类

### 输入

- **输入来源**：用户手动输入（表格）
- **输入格式**：每行一个单位，字段：名称（文本）/ HP（正整数）/ ATK（正整数）/ DEF（正整数）
- **输入校验规则**：名称非空；HP/ATK/DEF 为 1~9999 的正整数

### 处理流程

```
输入 → 解析表格所有单位 → 校验合法性 → 两两组合 → 对每组合运行 simulateBattle() → 收集 BattleResult[] → 渲染结果矩阵
```

### 输出

- **输出格式**：N×N 颜色矩阵，格子内显示"X回合胜"
- **输出展示方式**：内嵌在页面右侧，不需要导出

### 异常处理

- 输入为空时：按钮灰化，提示"请至少输入 2 个单位"
- 格式错误时：对应格子边框变红，"模拟"按钮灰化直到修正
- 超时/溢出时：超出 MAX_SIM_ROUNDS → 格子显示"平局"，不视为错误

### 状态持久化

- 是否需要保存历史记录：否（每次刷新重置）
- 是否需要从上次中断处恢复：否
