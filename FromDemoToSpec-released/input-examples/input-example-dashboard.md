## 必填区：所有原型通用

### A. 验证目标

- **要验证的核心假设是什么？**
  运营负责人打开仪表板后，能在 30 秒内不借助任何解释说出"今天哪个指标异常"——即当前的卡片排版、图表类型、高亮规则是否足够高效传达信息优先级。

- **验证成功的判据是什么？**
  3 位受众（运营/产品）在首次看到仪表板后，30 秒内都能正确指出预设的 1 个"异常数据"；且至少 2 人能说出"我最想看的数据在第一眼就找到了"。

- **不需要验证什么？**
  不需要验证：数据真实性、后端接入、权限控制、历史数据钻取、移动端布局。只验证桌面端视觉层面的信息传达效率。

---

### B. 核心交互

- **用户做什么操作？**
  打开仪表板（只读） → 扫视关键指标卡 → 切换时间粒度（日/周/月）→ 点击某图表查看趋势细节 → 切换数据集模拟不同场景

- **系统如何响应？**
  切换时间粒度：重新渲染所有图表和数值，动画过渡 → 点击图表：展开趋势折线图浮层 → 切换数据集：注入预设模拟数据，触发异常高亮规则

- **交互循环（用箭头表示）**
  ```
  打开仪表板 → 扫视指标卡 → 发现异常高亮 → 点击展开细节 → 调整时间粒度 → 再次扫视 → （反复）
  ```

---

### C. 数据对象

| 对象名 | 关键属性 | 与其他对象的关系 |
|--------|----------|------------------|
| MetricCard | id, label, value, unit, trend(vs昨日%), anomalyThreshold, isAnomaly | 独立卡片；isAnomaly 触发高亮 |
| ChartData | metricId, timeGranularity, dataPoints[]{date, value} | 为 MetricCard 提供趋势数据；按 timeGranularity 切换 |
| Dashboard | cards[], layout[], activeGranularity, activeDataset | 持有所有 MetricCard；控制布局和当前数据集 |
| DataSet | id, name, cards[]{metricId, value, trend} | 预设模拟数据（正常/异常/极端）；注入 Dashboard |

---

### D. 状态与流程

- **系统有哪些状态？**
  `loading`（初始化数据）→ `viewing`（正常浏览）→ `detail`（某图表展开）→ `switching`（切换粒度/数据集，过渡动画中）

- **状态之间如何切换？**
  ```
  loading    → （数据注入完成）                  → viewing
  viewing    → （点击图表卡片）                  → detail
  detail     → （点击关闭 / 点击空白区）          → viewing
  viewing    → （切换时间粒度 / 数据集按钮）      → switching（~300ms 后自动回 viewing）
  ```

- **初始状态**：`loading` → 立即变 `viewing`，默认数据集：正常场景，时间粒度：日
- **结束状态**：无终止状态（仪表板持续运行），重置按钮回到默认数据集

---

### E. 核心规则

```
- 规则1：异常判定：value 超出 anomalyThreshold（±20% 为默认阈值）→ 卡片红色高亮边框
- 规则2：趋势指示：trend > 0 显示绿色上箭头，trend < 0 显示红色下箭头（方向颜色可调参数控制）
- 规则3：布局优先级：异常卡自动上浮到第一排（可调参数控制是否开启）
- 规则4：时间粒度切换时，图表数据重新计算，指标卡数值显示该粒度的最新一期
- 边界条件（必须不能发生的事）：
    - 不允许同时展开 2 个图表细节浮层
    - 不允许在 switching 状态触发新的切换（防止动画冲突）
    - anomalyThreshold 为 0 时禁用异常高亮，不报错
- 特殊情况处理：
    - 数据集中某指标缺失：显示"—"占位，不崩溃
    - 全部指标都异常：最多高亮前 3 张（避免"全红失去重点"）
```

---

### F. UI 骨架

- **整体布局**：
  顶部工具栏（标题 + 时间粒度切换 + 数据集切换 + 重置）+ 中央卡片网格（3列，卡片响应式）+ 调试面板（右下浮层）

- **关键视觉元素位置**：
  - 指标卡：卡片内显示指标名/当前值/单位/趋势箭头/趋势百分比；异常时红色边框+叹号角标
  - 图表细节浮层：全屏遮罩+居中卡片，内含折线图和数据表
  - 时间粒度按钮：顶部工具栏，日/周/月三个切换按钮
  - 数据集选择：下拉菜单，供验证者快速切换"正常/异常/极端"场景

- **信息展示优先级**：
  1. 异常卡（最大视觉权重，红色高亮）
  2. 核心指标当前值（大字号）
  3. 趋势方向和幅度（箭头+百分比）
  4. 图表趋势（点击后展开）

---

### G. 可调参数（调试面板数据源）

| 参数名 | 默认值 | 可调范围 | 说明 |
|--------|--------|----------|------|
| ANOMALY_THRESHOLD_PCT | 20 | 5~50 | 异常判定阈值（波动超过此%触发高亮） |
| MAX_HIGHLIGHTED_CARDS | 3 | 1~9 | 最多同时高亮的异常卡数量 |
| AUTO_SORT_ANOMALY | true | true/false | 异常卡是否自动排到第一排 |
| TREND_UP_COLOR | green | green/blue/gray | 上升趋势颜色（部分业务"下降才是好事"） |
| CHART_ANIMATION_MS | 300 | 0~1000 | 图表切换动画时长（0=无动画） |

## 选填区：数据可视化/仪表板类

### 指标定义

| 指标 ID | 名称 | 单位 | 正常场景值 | 异常场景值 | 图表类型 |
|---------|------|------|-----------|-----------|---------|
| dau | 日活跃用户 | 万人 | 12.5 | 8.2（-34%，触发异常） | 折线图 |
| arpu | 人均收入 | 元 | 3.2 | 3.5（+9%，正常） | 折线图 |
| retention_d1 | 次日留存 | % | 42 | 38（-10%，边界） | 折线图 |
| pay_rate | 付费率 | % | 5.8 | 7.1（+22%，触发异常） | 折线图 |
| session_len | 平均会话时长 | 分钟 | 18 | 18（正常） | 折线图 |

### 预设数据集

| 数据集名 | 描述 | 异常指标 |
|---------|------|---------|
| normal | 所有指标正常波动 | 无 |
| anomaly_dau | DAU 单日暴跌 34% | dau |
| double_anomaly | DAU 下滑 + 付费率异常上升（可能刷量） | dau + pay_rate |

### 可视化设计决策

```
判断优先级：
1. 只有 1 个异常 → 验证单异常高亮是否被第一眼发现
2. 2 个异常 → 验证多异常场景下优先级排布是否合理（高亮排序）
3. 0 个异常 → 验证"全绿"状态的阅读效率（不应看起来"有什么问题没显示"）
```

### 布局说明

```
Row 1（首要）：dau（大卡，跨2列） + pay_rate（标准卡）+ retention_d1（标准卡）
Row 2（次要）：arpu（标准卡）+ session_len（标准卡）+ [预留空卡]
```
