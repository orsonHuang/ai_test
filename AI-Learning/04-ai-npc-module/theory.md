# 04 — 综合实战：AI同伴阿尔法完整模块

> 交付目标：一个"会说话、会查资料、会做事、有记忆"的完整AI-NPC模块

---

## 1. 模块定位

**模块名称：** AI同伴阿尔法 — 基地管家完整版

**核心体验：**
- 阿尔法是玩家的"第二双手"——不只是聊天对象，而是能查数据、触发事件、记住你做过什么的伙伴
- 对话有连贯性（记住3轮前的对话内容）
- 回答基于世界观设定（RAG）
- 能主动执行操作（Function Calling + ReAct）

**一句话设计目标：**
> 玩家跟阿尔法对话时，感觉"他真的在帮我管理基地"，而不是"他在给我念台词"。

## 3. 新增能力：记忆系统

### 为什么需要记忆？

01-03的阿尔法每次对话都是"新人"——他不记得你上一轮说了什么，不记得你修了通讯天线，不记得沙尘暴刚过。

游戏NPC需要三种记忆：

| 记忆类型 | 内容 | 用途 |
|----------|------|------|
| 短期记忆 | 最近5轮对话摘要 | 保证对话连贯性，不会重复问同一个问题 |
| 长期记忆 | 玩家的关键决策和事件 | "你修了通讯天线" → 下次对话会提及 |
| 关系记忆 | 玩家与NPC的关系变化 | 帮过阿尔法 → 语气更信任；忽略警告 → 更焦虑 |

### 实现方式

- 矩期记忆：每轮对话结束后，把用户输入和阿尔法回复都存入列表，取最近5轮压缩成摘要
- 长期记忆：提取对话中的关键事件（"修复了XXX"、"完成了XXX"），存为独立条目
- 这两种记忆在每次新对话时都嵌入 System Prompt

## 4. 完整对话流程

```
玩家说："阿尔法，上次修的天线还好吗？"
    ↓
Step 1: 检查记忆
    长期记忆里有"玩家在第5天修复了通讯天线"
    短期记忆里有最近几轮对话摘要
    ↓ 嵌入Prompt
Step 2: 知识库检索（RAG）
    搜索"通讯天线" → 检索到设施设定文档块
    ↓ 增强Prompt
Step 3: 工具调用决策（Function Calling）
    LLM判断：需要查天线当前状态
    → 调用 get_facility_status("comm_antenna")
    → 返回：状态=正常，效率=1.0
    ↓ 观察结果
Step 4: 综合生成回复
    基于记忆（你修的）+ 知识库（设定）+ 工具结果（当前状态）
    → "你第5天亲手修的天线运行正常，效率100%。信号增强器可以升级了。"
```

## 6. 考核标准

04阶段不再是答题，而是**项目交付评审**：

1. **策划案完整性** — 模块定位、体验目标、技术架构是否清晰？
2. **代码可运行** — alpha-full.py 能跑，不报错，输出格式正确？
3. **记忆生效** — 连续对话时，阿尔法能引用之前的对话内容？
4. **RAG生效** — 问世界观问题时，引用来源标注正确？
5. **工具调用生效** — 需要查数据时，阿尔法主动调工具而非瞎编？
6. **5分钟演示** — 能向别人展示"阿尔法记住我修了天线 + 基于设定回答 + 主动查数据"三个核心能力？
