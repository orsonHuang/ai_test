## 01 — AI-NPC 基础：LLM + Prompt + 体验设计

### 理论速通
- [ ] LLM 是什么：Transformer、Token、Context Window、Temperature
- [ ] 主流模型：GPT-4o、Claude、DeepSeek、Qwen，各模型对游戏文本生成的偏好
- [ ] Prompt Engineering 核心技巧
  - 角色卡设定（人格、背景、说话风格、禁忌）
  - Few-shot 对话示例
  - Chain-of-Thought 让NPC"有理由"地回应
  - 结构化输出（JSON/对话分支/动作指令）
- [ ] AI-NPC 与传统NPC的区别
  - 传统：状态机 + 对话树
  - AI：开放式、上下文记忆、能动态反应
- [ ] AI-NPC 体验设计原则
  - 性格一致性
  - 可引导性（不能让玩家跑题）
  - 安全性（不破坏世界观、不暴力、不瑟瑟）
  - 成本控制与响应速度

### 动手项目：AI-NPC 对话原型
**为《遥远行星：建造师》设计第一个NPC：基地AI助手「阿尔法」**
- 角色：飞船基地的AI管家，负责新手引导与日常任务提醒
- 功能：
  - 根据玩家当前状态给出不同对话
  - 支持自然语言询问基地设施
  - 输出带有情绪的文本 + 建议动作
- 输出：一个可运行的Python脚本，输入玩家状态，输出NPC对话
- 文件：`01-ai-npc-basics/practice/alpha-npc.py`

## 02 — 游戏知识库：RAG + NPC记忆与世界观

### 理论速通
- [ ] RAG 核心：检索增强生成，让AI"读"游戏设定
- [ ] Embedding 模型：把文本变成向量
- [ ] 向量数据库：ChromaDB、FAISS
- [ ] 文档分块：世界观、角色、物品、任务、规则文档怎么处理
- [ ] NPC长期记忆：对话历史、玩家关系、事件记忆
- [ ] 记忆压缩：把长对话总结成"人物关系摘要"

### 动手项目：游戏知识库 + NPC问答
**为《遥远行星：建造师》构建可查询的世界观知识库**
- 内容：星球设定、种族、资源、建筑、NPC背景
- 功能：玩家或策划用自然语言查询设定
- 进阶：NPC能基于自己的"记忆"回答玩家问题
- 文件：`02-game-knowledge/practice/world-knowledge-rag.py`

- [ ] 模拟经营中的AI应用

## 04 — 综合实战：完整AI-NPC模块交付

### 目标
为《遥远行星：建造师》交付一个**可独立运行、可演示**的AI-NPC模块。

### 候选模块（任选其一）

**模块A：AI 同伴/副官**
- 跟随玩家，能对话、给建议、执行任务
- 有个性、有记忆、有情绪变化
- 能调用游戏内简单指令（如"检查资源""扫描区域"）

**模块B：AI 殖民者/NPC居民**
- 每个居民有独立人格、日程、需求
- 与玩家对话时基于当前状态（饥饿、工作、心情）
- 能触发事件、生成任务

**模块C：AI 世界演化系统**
- 根据玩家行为和规则模拟星球生态/ colony 发展
- 生成新闻、公告、变化描述
- 与玩家自然语言互动，解释世界变化

**模块D：AI 任务/剧情生成器**
- 输入玩家当前进度和世界状态
- 生成符合世界观和风格的支线任务
- 输出可配置的游戏数据（JSON）

### 交付标准
- [ ] 有清晰的策划案（文档）
- [ ] 有可运行的代码原型
- [ ] 有明确的输入输出接口
- [ ] 有3个以上的测试用例
- [ ] 有成本/性能评估
- [ ] 可5分钟内向他人演示
