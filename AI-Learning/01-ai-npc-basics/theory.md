# 01 — AI-NPC 基础：LLM + Prompt + 体验设计

> 目标：理解 LLM 底层原理，掌握 Prompt 工程，能设计首个 AI-NPC 对话体验

---

## 1. LLM 是什么？（游戏策划需要知道的版本）

**LLM = 大语言模型 = 超级文本预测器**

给它一段文本，它预测下一个最可能出现的词。经过海量训练后，模型涌现出了理解、推理、角色扮演、讲故事的能力——这正是 AI-NPC 的技术基础。

### 关键术语

| 术语 | 通俗解释 | 对 AI-NPC 的影响 |
|------|----------|------------------|
| **Token** | 模型处理的最小文本单位，中文约1-2字 | 对话越长越贵，要控制上下文长度 |
| **Context Window** | 模型一次能处理的最大Token数 | 决定 NPC 能"记住"多少对话历史 |
| **Temperature** | 输出随机性，0=保守，1=有创造力 | 严肃NPC调低，搞怪NPC可调高 |
| **System Prompt** | 给AI设定的"角色剧本" | 决定 NPC 的性格、语气、世界观 |
| **Few-shot** | 给几个例子让AI模仿 | 快速让NPC学会特定说话风格 |

### 一句话理解 Transformer

每个词在理解上下文时，会通过"注意力机制"判断哪些词更重要。所以模型能区分：
- "苹果很好吃" → 苹果是水果
- "苹果很好玩" → 苹果是手机/品牌

## 2. AI-NPC 与传统 NPC 的区别

### 传统 NPC
```
玩家选择 → 查对话树 → 返回固定文本
```
- 优点：可控、可测试、成本低
- 缺点：对话重复、无法灵活应对玩家

### AI-NPC
```
玩家输入 + NPC人设 + 世界观 + 记忆 → LLM → 动态生成回应
```
- 优点：自然、沉浸、可创造涌现体验
- 缺点：需要Prompt设计、成本、可能"出戏"

### 设计核心：把"自由度"关在"世界观+角色"的笼子里

AI-NPC 不是让AI随便说，而是让它在：
- 固定的人设下
- 固定的世界观中
- 有记忆和情绪的上下文中
**自然地说出符合角色的话**。

## 3. Prompt Engineering 核心技巧

### 3.1 角色卡设计（System Prompt）

一个完整的角色卡应包含：

```
你是一位名为[名字]的NPC，存在于[游戏名]的世界中。

【基本身份】
- 角色：...
- 年龄/性别/种族：...
- 所在地点：...

【性格】
- 3个核心性格词
- 说话风格

【背景故事】
- 100字以内

【知识边界】
- 你知道什么
- 你不知道什么（严禁透露/讨论）

【对话规则】
- 禁止使用现代词汇
- 玩家问你[敏感话题]时，你要[特定反应]
- 每次回复控制在50字以内
- 保持角色一致性，不要跳出世界观

【当前状态】
- 情绪：...
- 正在做什么：...
- 对玩家的态度：...
```

### 3.2 Few-shot 示例

给AI看几个对话例子，它立刻学会风格：

```
示例1：
玩家："你是谁？"
阿尔法："我是阿尔法，基地AI管家。系统已运行3287天。你的氧气储备还剩12小时，建议优先处理。"

示例2：
玩家："今天有什么任务？"
阿尔法："侦测到东南区域有废弃信号。分析显示：60%概率为旧时代物资，40%为干扰。建议派遣侦察机器人。"

现在请按这个风格回答：
玩家："我累了"
```

### 3.3 Chain-of-Thought（让NPC"有理由"地回应）

```
玩家说：[输入]

在回复前，请按以下步骤思考：
1. 分析玩家意图
2. 判断这个请求是否符合我的角色/知识边界
3. 决定我的情绪反应
4. 生成符合角色和当前状态的回复

请直接输出回复，不要输出思考过程。
```

### 3.4 结构化输出

让NPC回复不仅是对话，还可以带动作指令：

```
请按 JSON 格式输出：
{
  "dialogue": "NPC说的话",
  "emotion": "当前情绪",
  "action": "NPC做的动作或游戏内触发的事件",
  "hint": "给玩家的暗示或下一步引导"
}
```

## 4. AI-NPC 体验设计原则

### 4.1 性格一致性
- 同一个NPC在不同对话中应该保持一致的语气和价值观
- 避免今天冷酷明天热情（除非剧情刻意安排）

### 4.2 可引导性
- 玩家可能会故意问离题的问题（"你支持哪个足球队？"）
- NPC要能把话题拉回游戏世界

### 4.3 边界控制
- 不说破坏世界观的内容
- 不现代、不Meta、不涉政、不暴力、不瑟瑟
- 对未知问题，用角色身份搪塞过去

### 4.4 成本与性能
- 对话Token越少越便宜
- 系统提示词可以精简，保持核心人设
- 不是所有对话都需要AI，简单对话可仍用固定模板

## 5. API 调用基础（Python）

```python
from openai import OpenAI

client = OpenAI(api_key="your-key")

response = client.chat.completions.create(
    model="gpt-4o",  # 或 deepseek-chat, claude-3-5-sonnet
    messages=[
        {"role": "system", "content": "你是..."},
        {"role": "user", "content": "玩家说的话"}
    ],
    temperature=0.7,
    max_tokens=200
)

reply = response.choices[0].message.content
print(reply)
```
